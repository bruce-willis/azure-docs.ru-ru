# <a name="azure-functions-runtime-overview-preview"></a>Обзор среды выполнения Функций Azure (предварительная версия)

Среда выполнения Функций Azure (предварительная версия) позволяет воспользоваться преимуществами простоты и гибкости модели программирования Функций Azure в локальной системе. Разработанная на базе той же технологии с открытым исходным кодом, что и Функции Azure, среда выполнения Функций Azure развертывается в локальной среде, предоставляя практически тот же интерфейс разработки, что и облачная служба.

![Портал предварительной версии среды выполнения Функций Azure][1]

Среда выполнения Функций Azure позволяет испытать Функции Azure перед фиксацией в облаке. Таким образом при миграции созданные вами ресурсы можно перенести в облако.  Среда выполнения также открывает новые возможности, например использование свободной вычислительной мощности локальных компьютеров для запуска однодневных пакетных процессов. Вы также можете использовать устройства в организации, чтобы выполнять условную отправку данных в другие системы (как локальные, так и облачные).

Среда выполнения Функций Azure состоит из двух компонентов:

* роль управления средой выполнения Функций Azure;
* рабочая роль среды выполнения Функций Azure.

## <a name="azure-functions-management-role"></a>Роль управления Функциями Azure

Компонент роли управления Функциями Azure предоставляет площадку для управления функциями в локальной среде. Этот компонент выполняет следующие задачи:

* Размещает портал управления для решения "Функции Azure", который аналогичен [порталу Azure](https://portal.azure.com). Портал обеспечивает согласованный порядок работы, что позволяет разрабатывать функции так же, как и на портале Azure.
* Распределяет функции между несколькими рабочими ролями функций.
* Предоставляет конечную точку публикации, чтобы публиковать функции непосредственно из Microsoft Visual Studio. Для этого скачивается и импортируется профиль публикации.

## <a name="azure-functions-worker-role"></a>Рабочая роль Функций Azure

Рабочие роли для решения "Функции Azure" развертываются в контейнерах Windows, в которых выполняется код функции.  Вы можете развернуть в своей организации несколько рабочих ролей. Это ключевой метод, позволяющий клиентам использовать свободную вычислительную мощность.  В многих организациях резервные вычислительные ресурсы обеспечиваются за счет того, что машины постоянно включены, но длительно не используются.

## <a name="minimum-requirements"></a>Минимальные требования

Чтобы приступить к работе со средой выполнения для решения "Функции Azure", установите Windows Server 2016 или Windows 10 Creators Update с доступом к экземпляру SQL Server.
