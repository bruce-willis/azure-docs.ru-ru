# <a name="transcription-guidelines"></a>Рекомендации по транскрибированию
Чтобы обеспечить максимальную эффективность использования текстовых данных для настройки акустической и языковой моделей, следует придерживаться рекомендаций по транскрибированию. Эти рекомендации зависят от языка.

## <a name="text-normalization"></a>Нормализация текста

Для оптимального использования в настройке акустической или языковой модели текстовые данные должны быть нормализованы, то есть преобразованы в стандартную однозначную форму, распознаваемую системой. В этом разделе описывается нормализация текста, выполняемая Пользовательской службой распознавания речи при импорте данных, и нормализация текста, которую должен выполнить пользователь, прежде чем импортировать данные.

## <a name="inverse-text-normalization"></a>Обратная нормализация текста

Процесс преобразования необработанного (неформатированного) текста обратно в форматированный текст, т. е. в текст с учетом регистра и знаков препинания, называется обратной нормализацией текста (ITN). Обратная нормализация текста выполняется для результатов, возвращаемых SAPI Microsoft Cognitive Services. Пользовательская конечная точка, развернутая с помощью Пользовательской службы распознавания речи, использует ту же обратную нормализацию текста, что и SAPI Microsoft Cognitive Services. Однако сейчас эта служба не поддерживает пользовательское обратное преобразование текста, поэтому новые термины, представленные в пользовательской языковой модели, не будут отформатированы в результатах распознавания.

## <a name="transcription-guidelines-for-en-us"></a>Рекомендации по транскрибированию для английского языка (en-US)

Текстовые данные, отправленные в эту службу, должны быть записаны в виде обычного текста только с использованием печатаемой кодировки ASCII. Каждая строка файла должна содержать текст только для одного высказывания.

Очень важно избегать использования знаков препинания Юникода. Они могут быть включены случайно при подготовке данных в текстовом редакторе или импорте данных с веб-страниц. Замените эти знаки соответствующими знаками ASCII. Например: 

| Знаки Юникод, которых следует избегать | Заменяющий знак ASCII |
|----- | ----- |
| “Hello world” (открывающие и закрывающие двойные кавычки) | "Hello world" (двойные кавычки) |
| John’s day (правая одинарная кавычка) | John's day (апостроф) |

### <a name="text-normalization-performed-by-the-custom-speech-service"></a>Нормализация текста, выполняемая Пользовательской службой распознавания речи

Эта служба выполняет описываемую ниже нормализацию текста для данных, импортированных в виде набора языковых данных или транскрипций для акустического набора данных. А именно:

*   перевод всего текста в нижний регистр;
*   удаление всех знаков пунктуации, кроме апострофов внутри слов;
*   расширение чисел в речевую форму, включая денежные суммы.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| Starbucks Coffee | starbucks coffee |
| "Holy cow!" said Batman. | holy cow said batman |
| "What?" said Batman's sidekick, Robin. | what said batman's sidekick robin |
| Go get - em! | go get em |
| i'm double jointed | i'm double jointed |
| 104 Main Street | one oh four main street |
| Tune to 102.7 | tune to one oh two point seven |
| Pi is about 3.14 | pi is about three point one four |
| It costs $3.14 | it costs three fourteen |

### <a name="text-normalization-required-by-users"></a>Нормализация текста, требуемая от пользователей

Чтобы эффективно использовать данные, перед импортом к ним должны быть применены следующие правила нормализации.

*   Сокращения должны записываться словами, которые отражают речевую форму.
*   Нестандартные числовые строк должны быть написаны словами.
*   Слова с неалфавитными знаками или смешанными буквенно-цифровыми знаками должны записываться так, как произносятся.
*   Общеупотребительные сокращения можно оставить в виде отдельной сущности без точек или пробелов между буквами, но другие сокращения должны быть записаны отдельными буквами, разделенными пробелами.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| 14 NE 3rd Dr. | fourteen northeast third drive |
| Dr. Strangelove | Doctor Strangelove |
| James Bond 007 | James Bond double oh seven |
| Ke$ha | Kesha |
| How long is the 2x4 | How long is the two by four |
| The meeting goes from 1-3pm | The meeting goes from one to three pm |
| my blood type is O+ | My blood type is O positive |
| water is H20 | water is H 2 O |
| play OU812 by Van Halen | play O U 8 1 2 by Van Halen |

## <a name="transcription-guidelines-for-zh-cn"></a>Рекомендации по транскрибированию для китайского языка (zh-CN)

Для текстовых данных, отправляемых в Пользовательскую распознавания службу речи, следует использовать кодировку **UTF-8 (включая метки порядка следования байтов)**. Каждая строка файла должна содержать текст только для одного высказывания.

Очень важно избегать использования полуширинных знаков препинания. Они могут быть включены случайно при подготовке данных в текстовом редакторе или импорте данных с веб-страниц. Замените эти знаки соответствующими полноширинными знаками. Например: 

| Знаки Юникод, которых следует избегать | Заменяющий знак ASCII |
|----- | ----- |
| “你好” (открывающие и закрывающие двойные кавычки) | "你好" (двойные кавычки) |
| 需要什么帮助? (вопросительный знак) | 需要什么帮助？ |

### <a name="text-normalization-performed-by-the-custom-speech-service"></a>Нормализация текста, выполняемая Пользовательской службой распознавания речи

Эта служба распознавания речи выполняет описываемую ниже нормализацию текста для данных, импортированных в виде набора языковых данных или транскрипций для акустического набора данных. А именно:

*   удаление всех знаков препинания;
*   расширение чисел до речевой формы;
*   преобразование полноширинных букв в полуширинные буквы;
*   перевод всех слов на английском языке в верхний регистр.

Ниже приведены некоторые примеры:

| Исходный текст | После нормализации |
|----- | ----- |
| 3.1415 | 三 点 一 四 一 五 |
| ￥3.5 | 三 元 五 角 |
| w f y z | W F Y Z |
| 1992年8月8日 | 一 九 九 二 年 八 月 八 日 |
| 你吃饭了吗 ? | 你 吃饭 了 吗 |
| 下午5:00的航班 | 下午 五点 的 航班 |
| 我今年21岁 | 我 今年 二十 一 岁 |

### <a name="text-normalization-required-by-users"></a>Нормализация текста, требуемая от пользователей

Чтобы эффективно использовать данные, _перед_ импортом к ним должны быть применены следующие правила нормализации.

*   Сокращения должны записываться словами, которые отражают речевую форму.
*   Эта служба не охватывает все числовые величины. Надежнее будет записывать числовые строки в речевой форме.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| 我今年21 | 我今年二十一 |
| 3号楼504 | 三号 楼 五 零 四 |

## <a name="transcription-guidelines-for-de-de"></a>Рекомендации по транскрибированию для немецского языка (de-DE)

Для текстовых данных, отправляемых в Пользовательскую распознавания службу речи, следует использовать только кодировку **UTF-8 (включая метки порядка следования байтов)**. Каждая строка файла должна содержать текст только для одного высказывания.

### <a name="text-normalization-performed-by-the-custom-speech-service"></a>Нормализация текста, выполняемая Пользовательской службой распознавания речи

Эта служба выполняет описываемую ниже нормализацию текста для данных, импортированных в виде набора языковых данных или транскрипций для акустического набора данных. А именно:

*   перевод всего текста в нижний регистр;
*   удаление всех знаков препинания, включая кавычки английской или немецкой кодировки ("test", 'test', "test„ или «test»);
*   исключение всех строк, содержащих специальные знаки, включая следующие: ^ ¢ £ ¤ ¥ ¦ § © ª ¬ ® ° ± ² µ × ÿ Ø¬¬
*   расширение чисел в речевую форму, включая денежные суммы;
*   надстрочные знаки допустимы только для a, o, u; другие будут заменены "th" или исключены.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| Frankfurter Ring | frankfurter ring |
| "Hallo, Mama!" sagt die Tochter. | hallo mama sagt die tochter |
| ¡Eine Frage! | eine frage |
| wir, haben | wir haben |
| Das macht $10 | das macht zehn dollars |


### <a name="text-normalization-required-by-users"></a>Нормализация текста, требуемая от пользователей

Чтобы эффективно использовать данные, перед импортом к ним должны быть применены следующие правила нормализации.

*   Десятичным разделителем должна быть запятая (","), а не точка ("."): 2,3%, а не 2.3%.
*   Разделителем между часами и минутами должно быть двоеточие (":"), а не точка ("."): 12:00 Uhr.
*   Такие сокращения, как "ca.", "bzw." не заменяются. Рекомендуется использовать полную форму, чтобы получить правильное произношение.
*   Пять основных математических операторов удаляются: +, -, \*, /.
 Мы рекомендуем заменить их речевыми формами: plus, minus, mal, geteilt.
*   То же относится к операторам сравнения (=, <, >,): gleich, kleiner als, grösser als.
*   Используйте дроби, например 3/4, в словесной форме (например, drei viertel вместо ¾).
*   Замените символ € словом Euro.


Ниже приведены некоторые примеры.

| Исходный текст | После нормализации пользователя | После нормализации системы
|--------  | ----- | -------- |
| Es ist 12.23Uhr | Es ist 12:23Uhr | es ist zwölf uhr drei und zwanzig uhr |
| {12.45} | {12,45} | zwölf komma vier fünf |
| 3 < 5 | 3 kleiner als 5 | drei kleiner als vier |
| 2 + 3 - 4 | 2 plus 3 minus 4 | zwei plus drei minus vier|
| Das macht 12€ | Das macht 12 Euros | das macht zwölf euros |
