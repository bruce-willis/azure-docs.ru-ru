# <a name="glossary-of-terms-for-custom-vision-service"></a>Глоссарий терминов Пользовательской службы визуального распознавания

Ниже приведены некоторые термины, используемые в Пользовательской службе визуального распознавания, и их значение.

## <a name="classifier"></a>Классификатор

Классификатор — модель, созданная с помощью Пользовательской службы визуального распознавания, на основе нескольких обучающих изображений. После завершения обучения нового классификатора вы получаете конечную точку оценки (HTTPS), которую можно добавить в приложение. Каждый классификатор создается в отдельном проекте. После входа вы можете просматривать все проекты.

## <a name="domain"></a>Домен

При создании проекта выбирается его домен. Домен оптимизирует классификатор конкретного типа объекта в изображениях. Например, если цель сценария заключается в определении изображений с яблочным пирогом и изображений с морковным пирогом, следует выбрать домен Food. Если вы не можете определить нужный домен, выберите домен Generic.

- **Домен Food.** Рассчитан на блюда из меню ресторана. Он не оптимизирован для распознавания фотографий отдельный фруктов и овощей. С этой целью следует использовать домен Generic.
- **Домен Landmark.** Рассчитан на распознавание естественных и искусственных ориентиров. Этот домен лучше всего работает, если на фотографии четко виден ориентир, даже если его немного заслоняет позирующая группа людей.
- **Домен Retail.** Рассчитан на распознавание изображений в каталогах товаров и на торговых веб-сайтах. Если требуется высокая точность при распознавании таких объектов, как платья, брюки, рубашки и т. д., следует использовать домен Retail.
- **Домен Adult.** Рассчитан на обозначение материалов для взрослых. Например, если вы хотите блокировать изображения людей в купальниках, этот домен дает возможность создать соответствующий пользовательский классификатор.
- **Домен General.** Рассчитан на самые разные задачи классификации изображений.

Модели, созданные с помощью **компактных доменов**, можно экспортировать с помощью функции экспорта итерации. Они оптимизированы для ограничений классификации в режиме реального времени на мобильных устройствах. Классификаторы, созданные с помощью компактного домена, могут быть точнее, чем с помощью стандартного домена с таким же объемом данных для обучения. Компромисс заключается в том, что они достаточно небольшие, чтобы запускать их локально в режиме почти реального времени. 

## <a name="training-image"></a>Обучающее изображение

Чтобы создать классификатор высокой точности, Пользовательской службе визуального распознавания требуется несколько обучающих изображений. Обучающее изображение — это фотография изображения, которое должна определять Пользовательская служба визуального распознавания. Например, чтобы определять апельсины, в Пользовательскую службу визуального распознавания необходимо передать несколько изображений апельсинов. Это позволит службе создать классификатор, который может распознать апельсины. Советуем использовать не менее 30 изображений каждого тега.

## <a name="iteration"></a>Итерация

При каждом обучении или повторном обучении классификатора вы создаете итерацию модели. Мы сохраняем несколько предыдущих итераций. Это позволяет сравнить прогресс со временем. Все ненужные итерации можно удалить. Следует помнить, что этот процесс необратим. Кроме того, также удаляются все изображения или изменения, используемые только в этой итерации. 

## <a name="workspace"></a>Рабочая область

Рабочая область содержит все обучающие изображения. В ней отражаются все изменения с последней итерации, например удаленные или добавленные изображения. В процессе обучения классификатора вы создаете его итерацию на основе изображений в рабочей области.

## <a name="tags"></a>Теги

С помощью тегов можно отмечать объекты на обучающих изображениях. Если вы создаете классификатор, определяющий собак и пони, на изображения с собаками следует добавить тег "собака", на изображения с пони — тег "пони", а на изображения с обоими этими животными — теги "собака" и "пони".

## <a name="evaluation"></a>Оценка

После обучения классификатора вы можете отправить любое изображение на оценку через автоматически созданную конечную точку HTTPS. Этот классификатор возвращает набор прогнозируемых тегов в порядке достоверности.

## <a name="predictions"></a>Прогнозирование

Классификатор хранит новые изображения, принятые для классификации. На основе этих изображений можно улучшить точность классификатора, правильно добавив теги на изображения с неверным предсказанием. Затем с помощью этих новых изображений можно повторно обучить классификатор.

## <a name="precision"></a>Precision

Какова вероятность правильного определения изображения классификатором при классификации? Из всех изображений, на основе которых выполнялось обучение классификатора (собаки и пони), какой процент модель определила правильно? 99 правильных тегов из 100 изображений дают точность 99 %.

## <a name="recall"></a>полнота;

Из всех изображений какое количество классификатор определил правильно? Отзыв 100 % означает, что если бы среди изображений, на основе которых выполнялось обучение, было бы 38 изображений собак, классификатор нашел бы их все.

## <a name="settings"></a>Параметры

Существует два типа параметров: уровня проекта и уровня пользователя.

- Параметры уровня проекта: 
  
  Параметры уровня проекта применяются к проекту или классификатору. К ним относятся следующие:

   - Домен проекта
   - Имя проекта
   - Описание проекта
   - Использование:
      - количество обучающих изображений;
      - количество созданных тегов;
      - количество созданных итераций.

- Параметры уровня пользователя: 
   - Ключи подписки (один для обучения и один для оценки и прогнозирования)
   - Использование:
      - количество созданных проектов;
      - количество выполненных вызовов API оценки и прогнозирования.
