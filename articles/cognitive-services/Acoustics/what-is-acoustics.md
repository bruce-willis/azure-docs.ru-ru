# <a name="what-is-project-acoustics"></a>Что такое Project Acoustics?
Плагин Project Acoustics Unity обеспечивает перекрытие, реверберацию и ориентацию в пространстве для проектов, предназначенных для традиционных экранов и экранов виртуальной реальности. Он предоставляет способ разработки игровой акустики, которая позволяющий "наложить" проектные планы на физическое проектирование волн.

## <a name="why-use-acoustics-in-virtual-environments"></a>Зачем использовать акустику в виртуальном окружении?
Люди воспринимают мир вокруг себя, используя аудиовизуальные подсказки. В виртуальных мирах сочетание пространственных звуков с акустикой увеличивает степень погружения пользователя. Акустическое средство, описанное здесь, анализирует виртуальные миры, чтобы создавать реалистичное акустическое моделирование, и поддерживает процесс моделирования после симуляции. Анализ включает в себя как геометрию, так и материалы для каждой поверхности в мире. Моделирование включает в себя такие параметры, как направление приема сигнала (перемещение), реверберация, время затухания, а также эффекты окклюзии и препятствий.

## <a name="how-does-this-approach-to-acoustics-work"></a>Как работает этот подход к акустике?
Система полагается на автономное вычисление виртуального мира, что позволяет проводить более сложную симуляцию, чем при анализе во время выполнения. При автономном вычислении создается таблица подстановки акустических параметров. Конструктор определяет правила, применяемые к параметрам во время выполнения. Корректировка этих правил позволяет достичь гиперреалистичных эффектов для обеспечения высокой эмоциональной интенсивности или создавать сюрреалистические сцены для фоновых аудиозвуков.

## <a name="design-process-comparison"></a>Сравнение процессов проектирования
Плагин Project Acoustics поддерживает новый процесс проектирования акустики в сценариях Unity. Чтобы объяснить этот новый процесс проектирования, давайте сравним его с одним из современных популярных подходов к акустике.

### <a name="typical-approach-to-acoustics-today"></a>Типичный современный подход к акустике
В типичном подходе к акустике вы сначала рисуете объемы реверберации:

![Представление конструктора](media/reverbZonesAltSPace2.png)

Затем настраиваете параметры для каждой зоны:

![Представление конструктора](media/TooManyReverbParameters.png)

Наконец, добавляете логику построения траектории лучей, чтобы получить правильную окклюзию и фильтрацию препятствий всей сцены, а также логику поиска маршрута для перемещения. На этот код могут потребоваться затраты во время выполнения. Он также имеет проблемы с плавностью за углами и пограничные случаи несимметричных сцен.

### <a name="an-alternative-approach-with-physics-based-design"></a>Альтернативный подход с физическим проектированием
Благодаря подходу, предоставляемому плагином Project Acoustics Unity, вы указываете размер и материалы статической сцены. Так как сцена — это трехмерное изображение и процесс не использует трассировку лучей, не нужно предоставлять упрощенную или безупречную акустическую решетку. Кроме того, не нужно отмечать сцену объемами реверберации. Плагин отправляет сцену в облако, где используется физическое проектирование волн. Результат интегрируется в ваш проект в качестве справочной таблицы, и его можно изменить для эстетических или игровых эффектов.

## <a name="requirements"></a>Требования
* Unity 2018.2+ для акустической обработки и Unity 5.2+ — для звукового проектирования и развертывания
* 64-разрядная версия Unity Editor для Windows
* Подписка пакетной службы Azure для акустической обработки
* Среда выполнения скриптов Unity — эквивалентная .NET 4.x

## <a name="platform-support"></a>Поддержка платформ
* Windows Desktop (x86 и AMD64)
* Универсальная платформа Windows (x86, AMD64 и ARM)
* Android (x86 и ARM64)
