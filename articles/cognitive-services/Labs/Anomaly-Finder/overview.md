# <a name="what-is-anomaly-finder"></a>Что такое средство для поиска аномалий

[!INCLUDE [PrivatePreviewNote](../../../../includes/cognitive-services-anomaly-finder-private-preview-note.md)]

Средство для поиска аномалий позволяет отслеживать данные с привязкой ко времени и обнаруживать в них аномалии благодаря методам машинного обучения, которые адаптируются к уникальным характеристикам и автоматически применяют правильную статистическую модель для любой отрасли, сценария и объема данных. Получив на вход данные временных рядов, API поиска аномалий возвращает данные о том, является ли некоторая точка данных аномалией, а также определяет для визуализации ее ожидаемое значение, верхний и нижний предел. Средство для поиска аномалий является предварительно созданной службой на основе искусственного интеллекта. Это означает, что для работы с ней не нужно хорошо разбираться в машинном обучении, достаточно лишь уметь использовать REST API. Благодаря этому разработка будет простой и гибкой, ведь служба работает с любыми данными временных рядов и легко встраивается в системы потоковой передачи данных. Средство для поиска аномалий поддерживает широкий диапазон вариантов использования — от финансовых средств для отслеживания мошенничества, краж, изменений на рынках и потенциальных инцидентов до мониторинга трафика устройств Интернета вещей с поддержанием анонимности. Это решение может приносить прибыль в составе службы, позволяющей получать сведения об изменениях в данных, затратах, рентабельности инвестиций или действиях пользователей.
Попробуйте в деле наш API поиска аномалий и получите новые сведения о своих данных. 

На основе этого API-интерфейса вы можете получить следующее:

* прогнозирование ожидаемых значений на основе исторических данных временных рядов;
* определение аномальных точек данных на основе исторических данных;
* вычисление границ для визуализации "нормального" диапазона значений.

## <a name="requirements"></a>Требования

- Минимальный набор входных данных временных рядов: не менее 13 точек данных для временных рядов без четкой периодичности или не менее 4 циклов точек данных для временных рядов с известной периодичностью. 
- Целостность данных: точки данных временных рядов должны располагаться через равные интервалы без пропусков. 

## <a name="identify-anomalies"></a>Выявление аномалий

API обнаружения аномалий возвращает сведения о том, являются ли конкретные точки данных аномалиями и предоставляет дополнительную информацию следующего содержания.
* Period — используемая API-интерфейсом периодичность для обнаружения точек аномалий.
* WarningText — возможные сведения о предупреждении.
* ExpectedValue — прогнозируемое значение по модели на основе обучения.
* IsAnomaly — признак, указывающий, являются ли точки данных аномалиями или нет.
* IsAnomaly_Neg — признак, указывающий, являются ли точки данных аномалиями в отрицательном направлении (провалами).
* IsAnomaly_Pos — признак, указывающий, являются ли точки данных аномалиями в положительном направлении (пиками).
* UpperMargin — сумма значений ExpectedValue и UpperMargin определяет верхнюю границу, в пределах которой точка данных еще считается нормальной.
* LowerMargin — разность значений ExpectedValue и LowerMargin определяет нижнюю границу, в пределах которой точка данных еще считается нормальной.

> [!Note]
> Значения UpperMargin и LowerMargin можно использовать для создания и визуализации диапазона обычных значений для этого временного ряда. 

## <a name="adjusting-lower-and-upper-bounds-in-post-processing-on-the-response"></a>Корректировка нижней и верхней границ при последующей обработки ответа

API обнаружения аномалий возвращает стандартную информацию о том, является ли точка данных аномалией, а по значениям ExpectedValue и UpperMargin или LowerMargin можно вычислить верхнюю и нижнюю границу. В большинстве случаев значения по умолчанию дают корректный результат. Но для некоторых сценариев нужны другие значения границ. Мы рекомендуем применять коэффициенты для значений UpperMargin или LowerMargin, чтобы динамически настраивать границы.
