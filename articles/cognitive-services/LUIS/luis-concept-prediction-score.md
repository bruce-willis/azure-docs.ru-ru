# <a name="prediction-score"></a>Оценка прогнозирования
Оценка прогнозирования означает степень достоверности результатов прогнозирования LUIS. 

Оценка прогнозирования обычно находится в диапазоне от нуля (0) до единицы (1). Пример оценки прогнозирования LUIS с высокой степенью достоверности — 0,99. Пример оценки прогнозирования LUIS с низкой достоверностью — 0,01. 

|Значение оценки|Достоверность|
|--|--|
|1|Точное соответствие|
|0.99|Высокая степень достоверности|
|0,01|Низкая степень достоверности|
|0|Точное несоответствие|

Если высказывание обладает низкой степенью достоверности, LUIS подчеркивает, что на странице **Intent** веб-сайта [LUIS](luis-reference-regions.md) идентифицировано **помеченное намерение**, выделенное красным. 

![Расхождение оценки](./media/luis-concept-score/score-discrepancy.png)

## <a name="top-scoring-intent"></a>Намерение с максимальной оценкой
Для каждого прогнозирования высказывание возвращает намерение с максимальной оценкой. Расхождение оценки — это числовое сравнение оценок прогнозирования. Две максимальные оценки могут иметь очень небольшое различие. LUIS никак не обозначает этот факт, за исключением возвращаемых оценок.  

Если вас беспокоит близость максимальных оценок, необходимо возвратить оценки для всех намерений. Вы можете либо добавить в два намерения высказывания, которые указывают различие намерений с помощью подбора и порядка слов, или воспользоваться приложением, которое вызывает LUIS, например чат-ботом, которое программным образом обработает два намерения с максимальной оценкой. 

Два намерения, которые оцениваются сходным образом, могут поменяться местами из-за недетерминированного обучения. Первая оценка может стать второй, а вторая может стать первой. Чтобы избежать этого, добавьте примеры фрагментов речи к каждому из намерений с наивысшими оценками с вариантами для выбора слов и контекстом, который будет отличать эти два намерения. Для двух намерений нужно добавить приблизительно одинаковое число примеров фрагментов речи. Общее правило для разделения для предотвращения изменения порядка намерений из-за обучение заключается в том, что разница в оценках должна составлять 15 %.

## <a name="return-prediction-score-for-all-intents"></a>Возврат оценки прогнозирования для всех намерений
Результат для теста или конечной точки может содержать все намерения. Такая конфигурация задается в [конечной точке](https://aka.ms/v1-endpoint-api-docs) с помощью пары "имя/значение" `verbose=true` в строке запроса. 

## <a name="review-intents-with-similar-scores"></a>Просмотр намерений с похожими оценками
Просмотр оценки для всех намерений позволяет не только определить правильное намерение, но и сделать так, чтобы оценка каждого следующего определенного намерения была значительно ниже для всех высказываний. 

Если у нескольких намерений есть близкие оценки прогнозирования, то на основе контекста высказывания LUIS может переключаться между намерениями. Чтобы устранить эту проблему, продолжайте добавлять высказывания с широким диапазоном контекстно-зависимых различий для каждого намерения.   

## <a name="e-exponent-notation"></a>Запись E (экспоненциальная запись)

Оценки прогнозирования можно записывать в форме экспоненциальной записи, *над* диапазоном 0–1, например `9.910309E-07`. Эта оценка свидетельствует об очень **маленьком** числе.

|Экспоненциальная запись оценки |Фактическая оценка|
|--|--|
|9.910309E-07|0,0000009910309|

## <a name="differences-with-predictions"></a>Различия в прогнозировании
Если при обучении той же модели в другом приложении вы получили другие оценки, это вызвано тем, что в обучении есть элемент случайности. Во-вторых, любое перекрытие высказывания более чем на одно намерение означает, что намерение с максимальной оценкой для того же высказывания может измениться в зависимости от обучения.

Если для обозначения достоверности для вашего чат-бота требуется конкретная оценка LUIS, следует использовать разницу оценок между двумя намерениями с максимальной оценкой. Это обеспечивает гибкость для различных вариантов обучения. 

## <a name="punctuation"></a>Пунктуация
В LUIS пунктуация является отдельным маркером. Для высказывания, которое заканчивается точкой, по сравнению с высказыванием, которое не является двумя отдельными фразами, можно получить два разных прогнозирования. Убедитесь, что модель обрабатывает пунктуацию либо согласно [примерам высказываний](luis-concept-utterance.md) (имеющих и не имеющих пунктуации), либо как [pattern} (luis-concept-patterns.md), где пунктуацию легче игнорировать с помощью специального синтаксиса: `I am applying for the {Job} position[.]`
