# <a name="intents-in-luis"></a>Намерения в LUIS

Намерение представляет задачу или действие, которое хочет выполнить пользователь. Эта цель или задача выражаются в [высказывании](luis-concept-utterance.md) пользователя.

Определите набор намерений, которые соответствуют действиям пользователей в вашем приложении. Например, в приложении для путешествий можно определить следующие намерения:

Намерения приложения для путешествий   |   Примеры высказываний   | 
------|------|
 BookFlight     |   "Book me a flight to Rio next week" ("Забронировать авиаперелет в Рио на следующей неделе") <br/> "Fly me to Rio on the 24th" ("Найти авиаперелет в Рио 24-го числа") <br/> "I need a plane ticket next Sunday to Rio de Janeiro" ("Мне нужен билет на самолет в Рио-де-Жанейро на следующее воскресенье")    |
 Greeting     |   "Hi" ("Привет") <br/>"Hello" ("Привет") <br/>"Good morning" ("Доброе утро")  |
 CheckWeather | "What's the weather like in Boston?" ("Какая сейчас погода в Бостоне?") <br/> "Show me the forecast for this weekend" ("Покажите мне прогноз на эту неделю") |
 None         | "Get me a cookie recipe" ("Рецепт печенья")<br>"Did the Lakers win?" ("Лейкерс победили?") |

Во всех приложениях используется предварительно определенное намерение "[None](#none-intent-is-fallback-for-app)", это резервное намерение. 

## <a name="prebuilt-domains-provide-intents"></a>Намерения от предварительно созданных доменов
Наряду с намерениями, которые вы определили, вы также можете использовать предварительно созданные намерения из одного из предварительно созданных доменов. Дополнительные сведения об изменении намерений из предварительно созданных доменов для использования в своих приложениях см. в разделе [Использование предварительно созданных доменов в приложениях LUIS](luis-how-to-use-prebuilt-domains.md).

## <a name="return-all-intents-scores"></a>Получение оценок для всех намерений
Высказывание назначается одному намерению. Когда LUIS получает высказывание в конечной точке, он возвращает одно намерение с максимальной оценкой для этого высказывания. Если вам необходимо получить оценки для всех намерений в высказывании, вы можете указать флаг `verbose=true` в строке запроса [вызова к конечной точке](https://aka.ms/v1-endpoint-api-docs) API. 

## <a name="intent-compared-to-entity"></a>Сравнение намерения и сущности
Намерение представляет действие, которое должен выполнить бот для пользователя, и основано на всем высказывании. Сущность представляет слова или фразы, содержащиеся в высказывании. Высказывание может иметь только одно намерение с максимальной оценкой и может иметь несколько сущностей. 

<a name="how-do-intents-relate-to-entities"></a> Создайте намерение в том случае, когда _намерение_ пользователя вызывает выполнения действия в вашем клиентском приложении, например вызов функции checkweather(). Затем создайте сущность, представляющую параметры, необходимые для выполнения действия. 

|Пример намерения   | Сущность | Сущность в примерах высказываний   | 
|------------------|------------------------------|------------------------------|
| CheckWeather | { "type": "location", "entity": "seattle" }<br>{ "type": "builtin.datetimeV2.date","entity": "tomorrow","resolution":"2018-05-23" } | What's the weather like in (Какая сейчас погода в) `Seattle` `tomorrow`? |
| CheckWeather | { "type": "date_range", "entity": "this weekend" } | Show me the forecast for (Покажите мне прогноз на) `this weekend` | 

## <a name="custom-intents"></a>Пользовательские намерения

[Высказывания](luis-concept-utterance.md) с похожей направленностью соответствуют одному намерению. Высказывание в вашем намерении может использовать любую [сущность](luis-concept-entity-types.md) в приложении, так как сущности не зависят от намерения. 

## <a name="prebuilt-domain-intents"></a>Намерения в предварительно созданных доменах

В [предварительно созданных доменах](luis-how-to-use-prebuilt-domains.md) есть намерения с высказываниями.  

## <a name="none-intent-is-fallback-for-app"></a>Намерение None является резервным намерением приложения
Намерение **None** является резервным намерением и используется для всех высказываний, которые не были отнесены к другим намерениям. Оно используется для обучения высказываний LUIS, которые не являются важными в домене (предметной области) приложения. На намерение **None** должно приходиться от 10 до 20 процентов от общего числа высказываний в приложении. Не оставляйте это намерение пустым. 

### <a name="none-intent-helps-conversation-direction"></a>Намерение None помогает определить направление диалога
Если для высказывания определяется намерение None и высказывание возвращается боту, то бот может задать дополнительные вопросы или предоставить пользователю меню с вариантами действий. 

### <a name="no-utterances-in-none-intent-skews-predictions"></a>Отсутствие высказываний в намерении None приводит к искажению прогнозирования
Если вы не добавили ни одно высказывание для намерения **None**, LUIS принудительно поместит высказывание, находящееся за пределами домена, в одно из намерений домена. Это приведет к искажению оценок прогнозирования, так как LUIS будет обучен ложному намерению для высказывания. 

### <a name="add-utterances-to-the-none-intent"></a>Добавление высказываний в намерение None
Намерение **None** создается и оставляется пустым намеренно. Заполните его высказываниями, которые не относятся к вашему домену. Для намерения **None** хорошо подходят высказывания, которые совершенно не связаны с вашим приложением и областью деятельности вашего приложения. Например, в приложении для путешествий к намерению **None** не могут относиться высказывания, связанные с путешествиями: с бронированием, выставлением счетов, питанием, проживанием, багажом и обслуживанием на борту. 

Высказывания какого типа следует оставить для намерения None? Начните с конкретных вопросов, на которые ваш бот не должен ответить, например, "What kind of dinosaur has blue teeth?" ("У какого динозавра синие зубы?") Это конкретный вопрос, никак не связанный с приложением для путешествий. 

### <a name="none-is-a-required-intent"></a>Намерение None является обязательным
Намерение **None** является обязательным и не может быть удалено или переименовано.

## <a name="negative-intentions"></a>Отрицательные намерения 
Если вы хотите определить положительные и отрицательные намерения, например, "Я **хочу** автомобиль" и "Я **не хочу** автомобиль", вы можете создать два намерения (одно положительное и одно отрицательное) и добавить соответствующие высказывания для каждого из них. Вы также можете создать одно намерение и пометить положительное и отрицательное понятия как сущности.  

## <a name="intent-balance"></a>Баланс намерений
Для каждого намерения домена приложения необходимо соблюдать баланс. Не создавайте одно намерение с 10 высказываниями и другое с 500 высказываниями. Это нарушает баланс. При наличии такой ситуации изучите намерение с 500 высказываниями и посмотрите, можно ли заключить большую часть этих высказываний в [шаблон](luis-concept-patterns.md). 

Необходимость поддержания баланса не относится к намерению **None**. Это намерение должно содержать 10 % от общего числа высказываний в приложении.

## <a name="intent-limits"></a>Ограничения намерений
Ознакомьтесь с [ограничениями](luis-boundaries.md#model-boundaries), чтобы понять, какое количество намерений можно добавить в модель. 

### <a name="if-you-need-more-than-the-maximum-number-of-intents"></a>Если требуемое количество намерений превышает максимально допустимое значение 
Во-первых, определите, не используется ли в вашей системе слишком много намерений. 

### <a name="can-multiple-intents-be-combined-into-single-intent-with-entities"></a>Затем определите, можно ли объединить несколько намерений в одно намерение с сущностями 
Если намерения слишком близки, то LUIS может испытывать трудности в том, чтобы различить их. Намерений должно быть достаточно для того, чтобы охватить основные задачи пользователя. При этом не стоит стараться охватить все направления вашего кода. Например, в приложении для путешествий BookFlight и FlightCustomerService могут быть отдельными намерениями, однако намерения BookInternationalFlight и BookDomesticFlight могут быть слишком похожими. Если в вашей системе необходимо их различать, используйте сущности или другую логику вместо намерений. 

### <a name="dispatcher-model"></a>Модель диспетчера
Ознакомьтесь с дополнительными сведениями об объединении приложений LUIS и QnA Maker с помощью [модели диспетчера](luis-concept-enterprise.md#when-you-need-to-combine-several-luis-and-qna-maker-apps). 

### <a name="request-help-for-apps-with-significant-number-of-intents"></a>Если намерений слишком много, обратитесь в службу поддержки
Если уменьшение числа намерений или разделение намерений на несколько приложений не помогает добиться цели, обратитесь в службу поддержки. Если ваша подписка Azure включает службы поддержки, обратитесь в службу [технической поддержки Azure](https://azure.microsoft.com/support/options/).
