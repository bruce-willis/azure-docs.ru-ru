## <a name="guidelines-for-azure-marketplace"></a>Рекомендации по Azure Marketplace  

### <a name="guidelines-for-creating-a-microsoft-id-to-manage-a-marketplace-account"></a>Рекомендации по созданию идентификатора Майкрософт для управления учетной записью Marketplace  
Если нескольким пользователям требуется доступ к одному идентификатору Майкрософт, указанному при создании учетной записи Marketplace, выполните следующую процедуру, чтобы создать корпоративную учетную записи. 

>[!IMPORTANT]
>Чтобы предоставить нескольким пользователям доступ к одной учетной записи Центра разработчиков Microsoft, корпорация Майкрософт рекомендует назначить роли для каждого из этих пользователей через Azure Active Directory (Azure AD). Чтобы получить доступ к этой учетной записи, пользователи должны выполнить вход со своими учетными данными Azure AD. При создании идентификатора Майкрософт укажите адрес электронной почты в домене, зарегистрированном на вашу организацию. Мы рекомендуем использовать адрес, который не назначен конкретному сотруднику. Например, `windowsapps@fabrikam.com`.  
>*   Дополнительные сведения см. в разделе [об использовании идентификатора Майкрософт в федеративном домене Azure AD](#issue-microsoft-id-in-an-azure-ad-federated-domain).  

*   Доступ к идентификатору Майкрософт следует ограничить минимально необходимым числом разработчиков. 
*   Настройте корпоративный список рассылки по электронной почте, включив в него всех пользователей, которым нужен доступ к учетной записи Центра разработчиков. Добавьте адрес электронной почты для отправки этого списка рассылки в сведения о безопасности. Все сотрудники, включенные в этот список, смогут по мере необходимости получать коды безопасности и управлять сведениями о безопасности для идентификатора Майкрософт. Если список рассылки создать невозможно, владелец персонального адреса электронной почты должен всегда оставаться на связи, чтобы предоставить полученный код безопасности.  
    *   Например, владельцу идентификатора поступают сообщения при добавлении новых сведений о безопасности в идентификатор Майкрософт или при входе с нового устройства с использованием этого идентификатора.  
*   Предоставьте рабочий номер телефона (без добавочных номеров), который доступен для ключевых членов вашей команды.  
*   В большинстве случаев следует обязать разработчиков использовать только доверенные устройства для входа в учетную запись Центра разработчиков. Доступ к доверенным устройствам должен быть у всех основных членов команды. Использование для доступа доверенных устройств частично избавляет от необходимости отправлять коды безопасности при доступе к учетной записи Центра разработчиков.  
*   Если доступ к учетной записи Центра разработчиков требуется для компьютеров без доверия, предоставьте доступ не более чем пяти разработчикам. Желательно, чтобы все компьютеры, с которых разработчики обращаются к этой учетной записи, находились в одном географическом и сетевом расположении.  
*   Регулярно просматривайте и проверяйте сведения о безопасности.  
    *   Чтобы просмотреть сведения о безопасности, откройте страницу "Параметры безопасности" по адресу [account.live.com/proofs/Manage](https://account.live.com/proofs/Manage).

Доступ к учетной записи Центра разработчиков следует осуществлять в основном с доверенных компьютеров. Доступ с доверенных компьютеров очень важен, так как существует ограничение на количество кодов, создаваемых для каждой учетной записи Центра разработчиков в неделю. Кроме того, использование доверенных компьютеров повышает безопасность и согласованность процедур входа. 
*   Дополнительные подробности и рекомендации по безопасности для учетных записей Центра разработчиков вы найдете на странице, посвященной созданию учетной записи разработчика, которая расположена по адресу [docs.microsoft.com/windows/uwp/publish/opening-a-developer-account](https://docs.microsoft.com/windows/uwp/publish/opening-a-developer-account).

#### <a name="issue-microsoft-id-in-an-azure-ad-federated-domain"></a>Проблема. Использование идентификатора Майкрософт в федеративном домене Azure AD  
Для корпоративной учетной записи можно настроить федерацию с Azure Active Directory. Если при создании идентификатора Майкрософт вы попытаетесь указать корпоративный адрес электронной почты, для которого настроена федерация с Azure AD, вы получите сообщение об ошибке. Получив такое сообщение, свяжитесь с ИТ-отделом вашей компании и убедитесь, что эта учетная запись включена в федерацию с Azure AD. Проблема с федерацией Azure AD для адресов электронной почты уже известна и корпорация Майкрософт работает над ее устранением.  
*   Дополнительные сведения об Azure AD см. на странице документации по Azure Active Directory по адресу [docs.microsoft.com/azure/active-directory](https://docs.microsoft.com/azure/active-directory).

Корпорация Майкрософт предлагает временное решение. Выполните следующие действия, чтобы создать адрес электронной почты в домене `outlook.com` и правило для пересылки сообщений.  
1.  Откройте страницу создания учетной записи и щелкните ссылку "Получить новый адрес электронной почты". 
    *   Чтобы получить новый идентификатор Майкрософт, откройте страницу создания учетной записи по адресу [signup.live.com/signup](https://signup.live.com/signup).  
2.  Создайте адрес электронной почты и введите пароль. Итак, у вас теперь есть новый идентификатор Майкрософт и новый почтовый ящик в домене `outlook.com`. Завершите процесс регистрации и создания учетной записи.  

    >[!IMPORTANT]
    >Для регистрации в центре разработчиков необходимо использовать адрес электронной почты или список рассылки, зарегистрированный в качестве идентификатора Майкрософт. Мы рекомендуем использовать именно список рассылки, чтобы избежать зависимости от конкретных пользователей. Если используемый адрес или список рассылки еще не зарегистрированы, зарегистрируйте их прямо сейчас.    

    >[!Important]
    >Если адрес электронной почты находится в корпоративном домене `Microsoft`, вы не сможете использовать его для регистрации в Центре разработчиков.  

3.  Завершив создание идентификатора Майкрософт с адресом электронной почты Outlook, войдите в почтовый ящик Outlook. Создайте правило пересылки сообщений электронной почты. Это правило должно перемещать все электронные письма из этого почтового ящика в Outlook на адрес электронной почты, созданный в вашем домене для управления учетной записью Marketplace.  
    *   Чтобы войти в почтовый ящик Outlook, откройте страницу Outlook по адресу [outlook.live.com/owa](https://outlook.live.com/owa).  
    *   Дополнительные сведения о правилах пересылки вы найдете в статье "Использование правил в Outlook Web App для автоматической пересылки сообщений в другую учетную запись" по адресу [support.office.com/article/ Use-rules-in-Outlook-Web-App-to-automatically-forward-messages-to-another-account-1433e3a0-7fb0-4999-b536-50e05cb67fed](https://support.office.com/article/Use-rules-in-Outlook-Web-App-to-automatically-forward-messages-to-another-account-1433e3a0-7fb0-4999-b536-50e05cb67fed).  

1.  Правило пересылки отправляет все электронные письма и другие сообщения, полученные в учетной записи электронной почты Outlook, на другой адрес электронной почты в домене, зарегистрированном на вашу компанию. Для аутентификации в Центре разработчиков и на Портале Cloud Partner используется адрес электронной почты `outlook.com`.
