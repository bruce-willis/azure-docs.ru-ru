# <a name="summary-of-customer-data-request-features"></a>Сводка о возможностях запроса данных клиента

Центр Интернета вещей Azure — это предназначенная для корпоративных клиентов облачная служба на базе REST API, которая обеспечивает безопасный двухсторонний обмен данными между миллионами устройств и секционированной службой Azure.

[!INCLUDE [gdpr-related guidance](../../includes/gdpr-intro-sentence.md)]

Администратор клиента назначает отдельным устройствам идентификатор устройства. Данные устройства основаны на присвоенном идентификаторе. Корпорация Майкрософт не сохраняет сведения и не имеет доступа к данным, которые позволяли бы сопоставить идентификаторы устройств с отдельными пользователями.

Многие из устройств, управляемых в Центре Интернета вещей, не являются пользовательскими, например офисный термостат или фабричный робот. Однако клиенты могут считать некоторые устройства такими, которые содержат персональные данные, и по своему усмотрению использовать собственные средства или методы отслеживания инвентаризации, связывающие устройства с частными лицами. Центр Интернета вещей хранит все данные, связанные с устройствами, и управляет ими как персональными данными.

Администраторы клиентов могут использовать портал Azure или REST API службы, чтобы выполнять информационные запросы путем экспорта или удаления данных, связанных с идентификатором устройства.

Если для переадресации сообщений устройства в другие службы используется функция маршрутизации службы Центра Интернета вещей, то запросы данных должны выполняться администратором клиента для каждой конечной точки маршрутизации, чтобы выполнить полный запрос для данного устройства. Дополнительные сведения см. в справочной документации каждой конечной точки. Дополнительные сведения о поддерживаемых конечных точках см. в руководстве [Конечные точки Центра Интернета вещей](iot-hub-devguide-endpoints.md).

Если вы используете функцию интеграции службы "Сетка событий Azure" службы Центра Интернета вещей, то запросы данных должны выполняться администратором клиента для каждого подписчика этих событий. Дополнительные сведения см. в статье [Реагирование на события в Центре Интернета вещей, используя службу "Сетка событий" для запуска действий (предварительная версия)](iot-hub-event-grid.md).

Если вы используете функцию интеграции службы Azure Monitor службы Центра Интернета вещей для создания журналов диагностики, то запросы данных должны выполняться администратором клиента в отношении сохраненных журналов. Дополнительные сведения см. в статье [Мониторинг работоспособности Центра Интернета вещей Azure и быстрая диагностика неполадок](iot-hub-monitor-resource-health.md).

## <a name="deleting-customer-data"></a>Удаление данных клиента

Администраторы клиента могут использовать колонку устройств Интернета вещей расширения Центра Интернета вещей для удаления устройства. Эта операция также удалит все данные, связанные с этим устройством.

Выполнять операции удаления для устройств можно также с помощью REST API. Дополнительные сведения см. в статье [Service - Delete Device](/rest/api/iothub/service/deletedevice) (Служба. Удаление устройства).

## <a name="exporting-customer-data"></a>Экспорт данных клиента

Администраторы клиента могут использовать на портале Azure операции копирования и вставки в колонке устройств Интернета вещей расширения Центра Интернета вещей для экспорта данных, связанных с устройством.

Выполнять операции экспорта для устройств можно также с помощью REST API. Дополнительные сведения см. в [Service - Get Device](/rest/api/iothub/service/getdevice) (Служба. Получение сведений об устройстве).

> [!NOTE]
> При использовании корпоративных служб корпорация Майкрософт создает некоторые данные, известные как создаваемые системой журналы. Некоторые создаваемые системой журналы Центра Интернета вещей недоступны и не подлежат экспорту администраторами клиента. Эти журналы содержат сведения о фактических действиях, выполненных в службе, и диагностические данные, связанные с отдельными устройствами.
