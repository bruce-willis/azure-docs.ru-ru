# <a name="delete-a-recovery-services-vault"></a>Удаление хранилища служб восстановления

В этой статье описывается, как удалить все элементы из хранилища служб восстановления, а затем удалить хранилище. Невозможно удалить хранилище служб восстановления, если оно зарегистрировано на сервере и хранит данные резервного копирования. Если вы пытаетесь удалить хранилище, но у вас не получается, это означает, что хранилище по-прежнему настроено на получение данных резервного копирования.

Сведения об удалении хранилища см. в разделе [Удаление хранилища с портала Azure](backup-azure-delete-vault.md#delete-a-vault-from-azure-portal). Если вы не хотите сохранять данные в хранилище служб восстановления и хотите удалить хранилище, см. раздел [Принудительное удаление хранилища](backup-azure-delete-vault.md#delete-the-recovery-services-vault-by-force). Если вы не знаете, что находится в хранилище, и хотите убедиться, что хранилище можно удалить, см. раздел [Удаление зависимостей хранилища и удаление самого хранилища](backup-azure-delete-vault.md#remove-vault-dependencies-and-delete-vault).

## <a name="delete-a-vault-from-azure-portal"></a>Удаление хранилища с портала Azure

Если хранилище служб восстановления открыто, перейдите ко второму шагу.

1. Откройте портал Azure и на панели мониторинга откройте хранилище, которое требуется удалить.

   Если на панели мониторинга не закреплено хранилище служб восстановления, в главном меню щелкните **Все службы** и в поле списка ресурсов введите **службы восстановления**. Как только вы начнете вводить символы, список отфильтруется соответствующим образом. Чтобы просмотреть список хранилищ в своей подписке, нажмите **Хранилища служб восстановления**.

   ![Создание хранилища служб восстановления — шаг 1](./media/backup-azure-delete-vault/open-recovery-services-vault.png) <br/>

   После этого отобразится список хранилищ служб восстановления. 

   ![выберите хранилище в списке](./media/backup-azure-delete-vault/choose-vault-to-delete-.png)

2. В списке выберите хранилище для удаления. Выберите это хранилище, чтобы открыть панель мониторинга.

    ![выберите хранилище, чтобы открыть его панель мониторинга](./media/backup-azure-delete-vault/contoso-bkpvault-settings.png)

3. Чтобы удалить хранилище, на панели мониторинга хранилища нажмите **Удалить**. Вам нужно будет подтвердить, что вы действительно хотите удалить хранилище.

    ![выберите хранилище, чтобы открыть его панель мониторинга](./media/backup-azure-delete-vault/click-delete-button-to-delete-vault.png)

    Если возникнет **ошибка удаления хранилища**, вы можете удалить зависимости из хранилища или использовать PowerShell для принудительного удаления хранилища. В следующих разделах описано, как выполнить эти задачи.

    ![Ошибка удаления хранилища](./media/backup-azure-delete-vault/vault-delete-error.png)


## <a name="delete-the-recovery-services-vault-by-force"></a>Принудительное удаление хранилища служб восстановления

Чтобы удалить хранилище служб восстановления принудительно, используйте PowerShell. Принудительное удаление означает, что вы навсегда удаляете хранилище служб восстановления и все данные резервного копирования. 

> [!Warning]
> Если вы удаляете хранилище служб восстановления с помощью PowerShell, вы должны быть уверены, что хотите навсегда удалить все данные резервного копирования в хранилище.
>

Удаление хранилища служб восстановления:

1. Войдите в учетную запись Azure.

   Войдите в подписку Azure с помощью команды `Connect-AzureRmAccount` и следуйте инструкциям на экране.

   ```powershell
    Connect-AzureRmAccount
   ```
   При первом использовании службы Azure Backup зарегистрируйте поставщик служб восстановления Azure в своей подписке с помощью команды [Register-AzureRmResourceProvider](/powershell/module/AzureRM.Resources/Register-AzureRmResourceProvider).

   ```powershell
    Register-AzureRmResourceProvider -ProviderNamespace "Microsoft.RecoveryServices"
   ```

2. Откройте окно PowerShell с правами администратора.

3. Удалите ограничения с помощью `Set-ExecutionPolicy Unrestricted`.

4. Выполните следующую команду, чтобы загрузить пакет клиента Azure Resource Manager с сайта chocolately.org.

    `iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))`

5. Установите клиент API Azure Resource Manager с помощью следующей команды.

   `choco.exe install armclient`

6. На портале Azure получите идентификатор подписки и соответствующее имя группы ресурсов для хранилища служб восстановления, которое вы хотите удалить.

7. В PowerShell выполните следующую команду, указав идентификатор подписки, имя группы ресурсов и имя хранилища служб восстановления. Эта команда удалит хранилище и все зависимости.

   ```powershell
   ARMClient.exe delete /subscriptions/<subscriptionID>/resourceGroups/<resourcegroupname>/providers/Microsoft.RecoveryServices/vaults/<recovery services vault name>?api-version=2015-03-15
   ```
8. Войдите в подписку на портале Azure и проверьте, что хранилище удалено.


## <a name="remove-vault-dependencies-and-delete-vault"></a>Удаление зависимостей хранилища и удаление самого хранилища

Чтобы вручную удалить зависимости хранилища, удалите конфигурацию между каждым элементом или сервером и хранилищем служб восстановления. Выполняя следующую процедуру, используйте меню **Элементы резервного копирования** (см. рисунок) для следующих элементов:

* Резервные копии Azure Storage (файлы Azure)
* Резервные копии SQL Server на виртуальной машине Azure
* Резервные копии виртуальных машин Azure
* Резервные копии агента служб восстановления Microsoft Azure

Используйте меню **Инфраструктура резервного копирования** (см. рисунок) для следующих элементов:

* Резервные копии Azure Backup Server
* Резервные копии System Center DPM

    ![выберите хранилище, чтобы открыть его панель мониторинга](./media/backup-azure-delete-vault/backup-items-backup-infrastructure.png)

1. На панели мониторинга хранилища прокрутите вниз до раздела "Защищенные элементы" и нажмите **Элементы резервного копирования**. В этом меню вы можете останавливать и удалять файловые серверы Azure, серверы SQL Server в виртуальных машинах Azure и виртуальные машины Azure. В этом примере мы удаляем данные резервного копирования с файлового сервера Azure.

    ![выберите хранилище, чтобы открыть его панель мониторинга](./media/backup-azure-delete-vault/selected-backup-items.png)

2. Выберите тип резервной копии, чтобы посмотреть все элементы этого типа.

    ![выбор типа резервной копии](./media/backup-azure-delete-vault/azure-storage-selected-list.png)

3. Нажмите правой кнопкой мыши на элемент в списке и в контекстном меню выберите **Остановить резервное копирование**.

    ![выбор типа резервной копии](./media/backup-azure-delete-vault/stop-backup-item.png) 

    Откроется меню Stop Backup (Остановка архивации).

4. В меню **Остановить резервное копирование** в области **Выбрать вариант** выберите пункт **Удалить данные резервного копирования**, введите имя элемента и щелкните **Остановить резервное копирование**.

    Введите имя элемента, чтобы подтвердить его удаление. Кнопка **Остановить архивацию** станет доступна после проверки элемента. Если вы сохраните данные, вы не сможете удалить хранилище.

    ![Удалить данные архивации](./media/backup-azure-delete-vault/stop-backup-blade-delete-backup-data.png)

    При необходимости можно указать причину удаления данных и добавить комментарии. Чтобы убедиться, что операция выполнена, проверьте ![удаление данных резервного копирования](./media/backup-azure-delete-vault/messages.png) в сообщениях Azure. <br/>
    После завершения операции служба отправляет сообщение об *остановке резервного копирования и удалении данных резервного копирования*.

5. Удалив элемент в списке, в меню **Элементы резервного копирования** щелкните **Обновить**, чтобы увидеть элементы в хранилище.

      ![Удалить данные архивации](./media/backup-azure-delete-vault/empty-items-list.png)

      Если в списке нет элементов, перейдите к панели **Основные компоненты** в меню хранилища служб восстановления. В полях **Архивные элементы**, **Серверы управления архивацией** или **Реплицированные элементы** не должно быть элементов. Если в хранилище по-прежнему есть элементы, вернитесь к шагу 3 и выберите другой список типов элементов.  

6. Когда на панели инструментов хранилища не останется элементов, щелкните **Удалить**.

    ![Удалить данные архивации](./media/backup-azure-delete-vault/vault-ready-to-delete.png)

7. Чтобы подтвердить удаление хранилища, нажмите кнопку **Да**.

    Хранилище удаляется, а на портале открывается меню **Новая служба** .

## <a name="what-if-i-stop-the-backup-process-but-retain-the-data"></a>Что будет, если я остановлю резервное копирование, но сохраню данные?

Если вы остановите резервное копирование, но случайно *сохраните* данные, нужно удалить данные резервного копирования, прежде чем удалять хранилище. Чтобы удалить данные архивации, сделайте следующее:

1. В меню **Элементы архивации** щелкните элемент правой кнопкой мыши и в контекстном меню выберите **Удалить данные архивации**.

    ![Удалить данные архивации](./media/backup-azure-delete-vault/delete-backup-data-menu.png)

    Откроется меню **Удаление данных архивации**.
2. В меню **Удаление данных архивации** введите имя элемента и нажмите кнопку **Удалить**.

    ![Удалить данные архивации](./media/backup-azure-delete-vault/delete-retained-vault.png)

    Удалив данные, перейдите к шагу 4c и продолжите процесс.
