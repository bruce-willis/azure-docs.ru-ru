# <a name="azure-search-service-rest-api-version-2017-11-11-preview"></a>REST API службы поиска Azure, версия 2017-11-11-Preview
В этой статье описывается версия `api-version=2017-11-11-Preview` REST API службы "Поиск Azure", предлагающая экспериментальные функции, которые пока недоступны вообще.

> [!NOTE]
> Функции предварительной версии доступны для тестирования и экспериментов с целью получения отзывов и могут изменяться. Мы не рекомендуем использовать предварительные версии API в рабочих приложениях.


## <a name="new-in-2017-11-11-preview"></a>Новые возможности в 2017-11-11-Preview

Функция [**автоматического завершения** ](search-autocomplete-tutorial.md) добавляется к существующему [API предложений](https://docs.microsoft.com/rest/api/searchservice/suggestions), расширяя возможности упреждающего ввода на панели поиска. Автоматическое завершение возвращает потенциальные условия запросов, которые пользователь может выбрать в качестве строки запроса для последующего поиска. API предложений возвращает фактические документы в ответ на частичные входные данные. Результаты поиска предоставляются немедленно и динамически изменяются по мере возрастания длины и точности входных данных.

[**Когнитивный поиск**](cognitive-search-concept-intro.md). Это новая информационная функция в службе "Поиск Azure", которая находит скрытые сведения в нетекстовых источниках и недифференцированном тексте, преобразуя их в текстовое содержимое с полными возможностями поиска в службе "Поиск Azure". Указанные ниже ресурсы представлены или изменены в предварительной версии REST API. Все другие интерфейсы REST API одинаковы для общедоступной и предварительной версий.

+ [Операции набора навыков (api-version=2017-11-11-Preview)](https://docs.microsoft.com/rest/api/searchservice/skillset-operations)

+ [Создание индексатора (api-version=2017-11-11-Preview)](https://docs.microsoft.com/rest/api/searchservice/create-indexer)

+ [Предопределенные навыки](cognitive-search-predefined-skills.md)

Все остальные REST API одинаковы независимо от того, как вы настраиваете версию API. Например, `GET https://[service name].search.windows.net/indexes/hotels?api-version=2017-11-11-Preview` и `GET https://[service name].search.windows.net/indexes/hotels?api-version=2017-11-11` (без `Preview`) функционально эквивалентны.

## <a name="other-preview-features"></a>Другие возможности предварительной версии

Возможности, объявленные в более ранних предварительных версиях, все еще находятся в общедоступной предварительной версии. При вызове API с более ранней предварительной версией API вы можете продолжать использовать эту версию или переключиться на `2017-11-11-Preview` без изменений ожидаемого поведения.

+ [CSV-файлы в индексировании больших двоичных объектов Azure](search-howto-index-csv-blobs.md), представленные в `api-version=2015-02-28-Preview`, остаются в предварительной версии. Эта функция является частью индексирования больших двоичных объектов Azure и вызывается путем указания параметра. Каждая строка в CSV-файле индексируется как отдельный документ.

+ [Массивы JSON в индексировании больших двоичных объектов Azure](search-howto-index-json-blobs.md), представленные в `api-version=2015-02-28-Preview`, остаются в предварительной версии. Эта функция является частью индексирования больших двоичных объектов Azure и вызывается путем указания параметра. Каждый элемент массива индексируется как отдельный документ.

+ [Параметр запроса moreLikeThis](search-more-like-this.md) для поиска документов, связанных с указанным документом. Эта компонент был доступен в более ранних предварительных версиях. 


## <a name="how-to-call-a-preview-api"></a>Вызов API предварительной версии

Более старые предварительные версии по-прежнему работают, но со временем устаревают. Если код вызывает `api-version=2016-09-01-Preview` или `api-version=2015-02-28-Preview`, эти вызовы остаются действительными. Однако только в последнюю версию предварительного просмотра будут добавлены усовершенствования. 

Следующий пример синтаксиса демонстрирует вызов предварительной версии API.

    GET https://[service name].search.windows.net/indexes/[index name]/docs?search=*&api-version=2017-11-11-Preview

Существуют различные версии службы поиска Azure. Дополнительные сведения см. в разделе [Версии API](search-api-versions.md).
