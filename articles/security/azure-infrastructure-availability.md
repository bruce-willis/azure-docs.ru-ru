# <a name="azure-infrastructure-availability"></a>Доступность инфраструктуры Azure
Azure предоставляет высокую доступность на основе обширной избыточности, достигаемой с помощью технологии виртуализации. Azure предоставляет различные уровни избыточности для максимальной доступности данных клиентов.

## <a name="temporary-outages-and-natural-disaster"></a>Временные сбои и стихийные бедствия
Команда обслуживания и инфраструктуры Microsoft Cloud разрабатывает, создает, обслуживает и улучшает защиту облачной инфраструктуры. Эта команда гарантирует, что инфраструктура Azure предоставляет высокий уровень доступности и надежности, высокую эффективность и интеллектуальную масштабируемость. Команда предоставляет более безопасное, частное и доверенное облако.

Источники бесперебойного питания и обширные запасы аккумуляторов обеспечивают непрерывную подачу питания в случае кратковременного сбоя. Аварийные генераторы служат резервными источниками питания в ситуации длительных перебоев с электричеством или планового технического обслуживания. На случай стихийного бедствия в центре обработки данных хранятся запасы топлива.

Скоростные и надежные оптоволоконные сети соединяют центры обработки данных с другими крупными центрами и пользователями Интернета. Вычислительные узлы размещают рабочие нагрузки ближе к пользователям, чтобы уменьшить задержку, обеспечить геоизбыточность и повысить общую устойчивость службы. Команда инженеров работает круглые сутки, чтобы гарантировать непрерывную доступность служб.

Корпорация Майкрософт обеспечивает высокий уровень доступности с помощью расширенного мониторинга и реагирования на инциденты, службы поддержки, а также резервного копирования при отработке отказа. Географически распределенные центры операций Майкрософт работают круглосуточно и без выходных. Сеть Azure является одной из крупнейших в мире. Оптоволоконная сеть распространения содержимого соединяет центры обработки данных и граничные узлы, чтобы обеспечить высокую производительность и надежность.

## <a name="disaster-recovery"></a>Аварийное восстановление
Azure обеспечивает устойчивость данных в двух расположениях. Вы можете выбрать расположение для резервного копирования данных. В обоих расположениях Azure постоянно поддерживает три работоспособных реплики данных.

## <a name="database-availability"></a>Доступность базы данных
Azure гарантирует, что база данных доступна в Интернете через интернет-шлюз с непрерывной доступностью базы данных. Средство мониторинга оценивает работоспособность и состояние активных баз данных с пятиминутным интервалом.

## <a name="storage-availability"></a>Доступность службы хранилища
Azure предоставляет хранилище через масштабируемую и надежную службу хранения, которая обеспечивает конечные точки подключения. Это гарантирует прямой доступ к службе хранилища для приложений. Служба хранения эффективно обрабатывает входящие запросы к хранилищу и сохраняет целостность транзакций.
