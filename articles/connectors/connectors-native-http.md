# <a name="call-http-or-https-endpoints-with-azure-logic-apps"></a>Вызов конечных точек HTTP или HTTPS с помощью Azure Logic Apps

С помощью Azure Logic Apps и соединителя протокола HTTP вы можете автоматизировать рабочие процессы, которые обмениваются данными с любой конечной точкой HTTP или HTTPS, настроив для этого соответствующие приложения логики. Например, можно отслеживать конечную точку службы для веб-сайта. Когда на этой конечной точке произойдет событие, например, сбой веб-сайта, будет запущен рабочий процесс приложения логики и выполнены указанные действия. 

В качестве первого шага в вашем рабочем процессе можно использовать триггер HTTP для регулярной проверки или *опроса* конечной точки. При каждой проверке триггер отправляет вызов или *запрос* к конечной точке. Ответ от конечной точки определяет, выполняется ли рабочий процесс приложения логики. Триггер передает любое содержимое из ответа действиям в приложении логики. 

При необходимости действия HTTP можно использовать как любой другой этап в рабочем процессе для вызова конечной точки. Ответ от конечной точки определяет, как выполнить оставшиеся действия рабочего процесса.

Если вы не знакомы с приложениями логики, ознакомьтесь со статьей [Что такое Azure Logic Apps](../logic-apps/logic-apps-overview.md).

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, <a href="https://azure.microsoft.com/free/" target="_blank">зарегистрируйтесь для получения бесплатной учетной записи Azure</a>. 

* URL-адрес для целевой конечной точки, которую требуется вызвать. 

* Базовые знания [создания приложений логики](../logic-apps/quickstart-create-first-logic-app-workflow.md).

* Приложение логики, из которого необходимо вызвать целевую конечную точку. [Создайте пустое приложение логики](../logic-apps/quickstart-create-first-logic-app-workflow.md) для запуска с использованием триггера HTTP. Чтобы использовать действие HTTP, запустите свое приложение логики с помощью триггера.

## <a name="add-http-trigger"></a>Добавление триггера HTTP

1. Войдите на [портал Azure](https://portal.azure.com) и откройте свое приложение логики в конструкторе приложений логики, если оно еще не открыто.

1. В поле поиска введите "HTTP" в качестве фильтра. В списке триггеров выберите нужный триггер **HTTP**. 

   ![Выбор триггера HTTP](./media/connectors-native-http/select-http-trigger.png)

1. Укажите [параметры и значения триггера HTTP](../logic-apps/logic-apps-workflow-actions-triggers.md##http-trigger), которые необходимо включить в вызов целевой конечной точки. Настройте периодичность, с которой триггер будет проверять целевую конечную точку.

   ![Ввод параметров триггера HTTP](./media/connectors-native-http/http-trigger-parameters.png)

   Дополнительные сведения о HTTP-триггере, параметрах и значениях см. в [справочнике по типам триггеров и действий](../logic-apps/logic-apps-workflow-actions-triggers.md##http-trigger).

1. Продолжите создание рабочего процесса приложения логики с действиями, выполняемыми при срабатывании триггера.

## <a name="add-http-action"></a>Добавление действия HTTP

[!INCLUDE [Create connection general intro](../../includes/connectors-create-connection-general-intro.md)]

1. Войдите на [портал Azure](https://portal.azure.com) и откройте свое приложение логики в конструкторе приложений логики, если оно еще не открыто.

1. На последнем шаге, в котором необходимо добавить действие HTTP, выберите **Новый шаг**. 

   В этом примере приложение логики запускается при срабатывании триггера HTTP в качестве первого шага.

1. В поле поиска введите "HTTP" в качестве фильтра. В списке действий выберите действие **HTTP**.

   ![Выбор действия HTTP](./media/connectors-native-http/select-http-action.png)

   Чтобы добавить действие между шагами, переместите указатель на стрелку между шагами. 
   Выберите появившийся знак "плюс" (**+**), а затем щелкните **Добавить действие**.

1. Укажите [параметры и значения действия HTTP](../logic-apps/logic-apps-workflow-actions-triggers.md##http-action), которые необходимо включить в вызов целевой конечной точки. 

   ![Ввод параметров для действия HTTP](./media/connectors-native-http/http-action-parameters.png)

1. Когда закончите, сохраните приложение логики. На панели инструментов конструктора нажмите кнопку **Сохранить**. 

## <a name="authentication"></a>Authentication

Чтобы настроить проверку подлинности, выберите **Показать расширенные параметры** внутри действия или триггера. Дополнительные сведения о доступных типах проверки подлинности для триггеров и действий HTTP см. в [справочнике по типам триггеров и действий](../logic-apps/logic-apps-workflow-actions-triggers.md#connector-authentication).
