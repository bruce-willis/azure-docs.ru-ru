# <a name="migrate-to-the-latest-version-of-azure-machine-learning-service"></a>Переход на последнюю версию службы "Машинное обучение Azure" 

**Если вы установили приложение Workbench (предварительная версия) учетная запись и (или) обладаете учетными записями для предварительного просмотра экспериментирования и управления моделями, воспользуйтесь этой статьей, чтобы выполнить переход на последнюю версию.**  Если у вас не установлена предварительная версия Workbench или нет учетной записи для экспериментирования и (или) управления моделями, выполнять переход не нужно.

## <a name="what-can-i-migrate"></a>Что можно перенести?
Большинство артефактов, созданных в первой предварительной версии службы "Машинное обучение Azure", хранятся в вашем локальном или облачном хранилище. Эти артефакты не исчезнут. Для перехода зарегистрируйте артефакты повторно в обновленной службе "Машинное обучение Azure". 

В следующих таблицах и статье объясняется, что можно выполнить с помощью существующих активов и ресурсов до или после перехода на последнюю версию службы "Машинное обучение Azure". В течение некоторого времени также можно использовать предыдущую версию и ресурсы ([см. сроки предоставления поддержки перехода](overview-what-happened-to-workbench.md#timeline)).

|Актив или ресурс из старой версии|Возможен ли переход?|Действия|
|-----------------|:-------------:|-------------|
|Модели машинного обучения (как локальные файлы)|Yes|Отсутствует. Работает как раньше.|
|Зависимости и схемы моделей (как локальные файлы)|Yes|Отсутствует. Работает как раньше.|
|Проекты|Yes|[Подключите локальную папку к новой рабочей области](#projects).|
|Журналы выполнения|Нет |[Загружаемые](#history) в течение некоторого времени.|
|Файлы подготовки данных|Нет |[Подготовьте набор данных любого размера](#dataprep) для моделирования, используя новый пакет SDK подготовки данных для службы "Машинное обучение Azure", или используйте Azure Databricks.|
|Целевые объекты вычислений|Нет |Зарегистрируйте их в новой рабочей области.|
|Зарегистрированные модели|Нет |Повторно зарегистрируйте модель в рабочей области.|
|Зарегистрированные манифесты|Нет |Отсутствует. В новой рабочей области манифесты больше не используются в качестве концепции.|
|Зарегистрированные изображения|Нет |Повторно создайте в новой рабочей области образ развертывания Docker.|
|Развернутые веб-службы|Нет |Отсутствует. Они будут продолжать работать в том же виде, в котором они были созданы, <br/>или [разверните их повторно, используя последнюю версию](#services).|
|Учетные записи для экспериментирования <br/>и управления моделями|Нет |[Создайте рабочую область](#resources) вместо этого.|
|Интерфейс командной строки и пакет SDK для машинного обучения|Нет |Используйте для последующей работы новые [интерфейс командной строки](reference-azure-machine-learning-cli.md) (CLI) и пакет [SDK](http://aka.ms/aml-sdk).|


Дополнительные сведения об [изменениях в этом выпуске](overview-what-happened-to-workbench.md)?

>[!Warning]
>Эта статья не предназначена для пользователей студии машинного обучения Azure. Она предназначена для пользователей службы "Машинное обучение Azure", которые установили приложение Workbench (предварительная версия) и (или) обладают учетными записями для экспериментирования и управления моделями.

<a name="resources"></a>

## <a name="azure-resources"></a>Ресурсы Azure

Ресурсы, такие как ваши учетные записи для экспериментирования, учетные записи для управления моделями и вычислительные среды машинного обучения, нельзя перенести в последнюю версию службы "Машинное обучение Azure". См. на [временной шкале](overview-what-happened-to-workbench.md#timeline), как долго продолжат работать ресурсы.

Начните работу с последней версией, создав рабочую область машинного обучения Azure на [портале Azure](quickstart-get-started.md). Панель мониторинга для рабочей области портала поддерживается только в браузерах Edge, Chrome и Firefox.

Эта новая рабочая область представляет собой ресурс службы верхнего уровня и позволяет использовать все новые возможности службы "Машинное обучение Azure". [Дополнительные сведения об этой рабочей области и архитектуре](concept-azure-machine-learning-architecture.md).

<a name="projects"></a>

## <a name="projects"></a>Проекты

Вместо помещения в рабочую область в облаке проекты в последней версии теперь являются каталогами на локальном компьютере. [См. схему последней архитектуры](concept-azure-machine-learning-architecture.md). 

Чтобы продолжить использовать локальный каталог, содержащий файлы и скрипты, укажите имя каталога в команде Python ["experiment.submit"](http://docs.microsoft.com/python/api/azureml-core/azureml.core.experiment.experiment?view=azure-ml-py) или с помощью команды интерфейса командной строки "az ml project attach".

Например: 
```python
run = exp.submit(source_directory = script_folder, script = 'train.py', run_config = run_config_system_managed)
```

<a name="services"></a>

## <a name="deployed-web-services"></a>Развернутые веб-службы

Чтобы перенести веб-службы, повторно разверните свои модели, используя новый пакет SDK или интерфейс командной строки для новых целевых объектов развертывания. Не нужно изменять исходный файл оценки, файлы зависимостей файлов модели, файл среды и файлы схем. 

В последней версии модели развертываются как веб-службы в [экземпляры контейнеров Azure](how-to-deploy-to-aci.md) (ACI) или в кластеры [службы Azure Kubernetes](how-to-deploy-to-aks.md) (AKS). 

Дополнительные сведения см. в следующих статьях:
+ [Развертывание в ACI](how-to-deploy-to-aci.md)
+ [Развертывание в AKS](how-to-deploy-to-aks.md)
+ [Руководство. Развертывание моделей с помощью службы "Машинное обучение Azure"](tutorial-deploy-models-with-aml.md)

Когда [поддержка предыдущего интерфейса командной строки закончится](overview-what-happened-to-workbench.md#timeline), управление веб-службами, первоначально развернутыми с помощью учетной записи управления моделями, станет невозможным. Но эти веб-службы продолжат работать, пока будет поддерживаться служба контейнеров Azure (ACS).

<a name="history"></a>

## <a name="run-history-records"></a>Записи журнала выполнения

Хотя добавление в существующие журналы выполнения, находящиеся в старой рабочей области, невозможно, имеющиеся журналы можно экспортировать с помощью предыдущего интерфейса командной строки. После [окончания поддержки предыдущего интерфейса командной строки](overview-what-happened-to-workbench.md#timeline) экспорт этих журналов выполнения станет невозможным.

Начните обучать свои модели и отслеживать журналы выполнения, используя новый интерфейс командной строки и новый пакет SDK. Чтобы узнать, как это делать, используйте [Руководство. Обучение моделей с помощью службы "Машинное обучение Azure"](tutorial-train-models-with-aml.md).

Для экспорта журналов выполнения с помощью предыдущего интерфейса командной строки:

```azurecli
#list all runs
az ml history list

#get details about a particular run
az ml history info

# download all artifacts of a run
az ml history download
```

<a name="dataprep"></a>

## <a name="data-preparation-files"></a>Файлы подготовки данных
Файлы подготовки данных невозможно перенести без помощи Workbench. Но можно подготовить набор данных любого размера для моделирования, используя новый пакет SDK подготовки данных для службы "Машинное обучение Azure" или Azure Databricks для больших наборов данных.  [Дополнительные сведения о получении пакета SDK для подготовки данных](how-to-data-prep.md).
