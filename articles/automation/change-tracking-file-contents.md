# <a name="view-contents-of-a-file-that-is-being-tracked-with-change-tracking"></a>Просмотр содержимого файла, который отслеживается с помощью отслеживания изменений

Отслеживание содержимого файла позволяет просматривать содержимое файла до и после изменения, если файл отслеживается с помощью службы отслеживания изменений. Эта функция сохраняет содержимое файла в учетную запись хранения после каждого изменения.

## <a name="requirements"></a>Требования

* Для хранения содержимого файла требуется стандартная учетная запись хранения с моделью развертывания Resource Manager. Не следует использовать учетные записи хранения классической модели развертывания и модели "Премиум". Дополнительные сведения об учетных записях хранения см. в статье [Учетные записи хранения Azure](../storage/common/storage-create-storage-account.md)

* К используемой учетной записи хранения может быть подключена только одна учетная запись службы автоматизации.

* [Отслеживание изменений](automation-change-tracking.md) включено в учетной записи службы автоматизации.

## <a name="enable-file-content-tracking"></a>Включение отслеживания содержимого файла

1. На портале Azure откройте учетную запись службы автоматизации и выберите **Отслеживание изменений**.
2. В верхнем меню нажмите **Изменить параметры**.
3. Выберите **Содержимое файла** и нажмите **Ссылка**. Откроется панель **Добавление расположения содержимого для отслеживания изменений**.

   ![enable](./media/change-tracking-file-contents/enable.png)

4. Выберите подписку и учетную запись хранения для хранения содержимого файла. Если вы хотите включить отслеживание содержимого файла для всех существующих отслеживаемых файлов, выберите **Вкл.** для пункта **Отправка содержимого файла для всех параметров**. Затем вы сможете изменить эту настройку для каждого пути к файлу.

   ![настройка учетной записи хранения](./media/change-tracking-file-contents/storage-account.png)

5. После включения отображаются учетная запись хранения и URI SAS. Срок действия URI SAS истекает через 365 дней. Их можно создать заново, нажав на кнопку **Создать повторно**.

   ![список ключей учетной записи](./media/change-tracking-file-contents/account-keys.png)

## <a name="add-a-file"></a>Добавление файла

Ниже приведены инструкции для включения отслеживания изменений в файле:

1. На странице **Изменение параметров** в разделе **Отслеживание изменений** выберите вкладку **Файлы Windows** или **Файлы Linux** и нажмите **Добавить**

1. Введите сведения о пути к файлу и выберите **True** для пункта **Отправка содержимого файла для всех параметров**. Этот параметр включает отслеживание содержимого файла только для этого пути к файлу.

   ![добавление файла Linux](./media/change-tracking-file-contents/add-linux-file.png)

## <a name="viewing-the-contents-of-a-tracked-file"></a>Просмотр содержимого отслеживаемого файла

1. После обнаружения изменений для файла или файла в пути они отображаются на портале. Выберите изменение файла из списка изменений. Откроется панель **Сведения об изменении**.

   ![список изменений](./media/change-tracking-file-contents/change-list.png)

1. На странице **Сведения об изменении** отображается стандартная информация о файле до и после изменения. Нажмите **Просмотр изменений содержимого файлов** в левом верхнем углу для просмотра содержимого файла.

  ![сведения об изменении](./media/change-tracking-file-contents/change-details.png)

1. Откроется новая страница с двумя столбцами содержимого файла до и после изменения. Вы можете нажать **Встроенный**, чтобы увидеть встроенное представление изменений.

  ![просмотр изменения файлов](./media/change-tracking-file-contents/view-file-changes.png)
