# <a name="viewing-and-analyzing-data-in-log-analytics"></a>Просмотр и анализ данных в Log Analytics
На портале Azure доступны две возможности анализа данных, которые хранятся в Log Analytics, и создания запросов на ad-hoc-анализ. Запросы, создаваемые с помощью этих порталов, могут использоваться для других функций, таких как предупреждения и панели мониторинга.

## <a name="log-analytics-page"></a>Страница Log Analytics
Откройте страницу Log Analytics из раздела **Журналы** в меню Log Analytics. Это новый интерфейс для работы с данными журнала и создания запросов. В статье [Начало работы с порталом аналитики](query-language/get-started-analytics-portal.md) можно ознакомиться с общими сведениями об этом портале и проверить его возможности.

Страница Log Analytics предоставляет некоторые улучшения интерфейса [поиска по журналам (классическим)](#log-search-classic).

* Несколько вкладок. Создание отдельных вкладок для работы с несколькими запросами.
* Расширенная визуализация. Разнообразие параметров для работы с диаграммами.
* Улучшенная технология Intellisense и автозаполнение языка.
* Выделение синтаксиса. Повышает удобство чтения запросов. 
* Обозреватель запросов. Предоставление доступа к сохраненным запросам и функциям.
* Представление схемы. Просмотр структуры данных для помощи в написании запросов.
* Общий доступ. Создание ссылок на запросы или их закрепление на любой общей панели мониторинга Azure.
* Смарт-аналитика. Определение пиков на диаграммах и быстрый анализ их причины.
* Выбор столбцов. Сортировка и группирование столбцов в результатах запросов.

> [!NOTE]
> Страница Log Analytics имеет те же функции, что и портал расширенной аналитики, который является внешним инструментом за пределами портала Azure. Портал расширенной аналитики по-прежнему доступен, но ссылки и другие справочные материалы к нему заменяются этой новой страницей.

### <a name="resource-logs"></a>Журналы ресурсов
Новые функции Log Analytics интегрируются с различными ресурсами Azure, например Виртуальными машинами. Это означает, что страницу Log Analytics можно открыть напрямую через меню мониторинга ресурсов без перехода в Azure Monitor и Log Analytics и потери контекста ресурса. Пока что **журналы** доступны не для всех ресурсов Azure, но они будут отображаться в меню портала для отдельных типов ресурсов.

Если открыть Log Analytics из меню определенного ресурса, записи журнала автоматически отобразятся только для этого ресурса.   Чтобы написать запрос на добавление других записей, вам нужно открыть его из меню Log Analytics или Azure Monitor.

Следующие параметры еще не доступны в представлении ресурса службы Log Analytics:

- Сохранить
- задание оповещения;
- Обозреватель запросов
- переключение на другую рабочую область или ресурс (пока что не планируется).


### <a name="firewall-requirements"></a>Требования к брандмауэру
Ваш браузер запрашивает доступ к следующим адресам, чтобы получить доступ к странице Log Analytics и порталу расширенной аналитики.  Если браузер обращается к порталу Azure через брандмауэр, необходимо разрешить доступ для этих адресов.

| URI | IP-адрес | порты; |
|:---|:---|:---|
| portal.loganalytics.io | Динамический | 80, 443 |
| api.loganalytics.io    | Динамический | 80, 443 |
| docs.loganalytics.io   | Динамический | 80, 443 |


## <a name="log-search-classic"></a>Поиск по журналам (классическим)
Откройте страницу поиска по журналам из раздела **Журналы (классические)** в меню Log Analytics или из **Log Analytics** в меню Azure Monitor. Это классическая страница для работы с запросами Log Analytics, на которой нет дополнительных возможностей [страницы Log Analytics](#log-analytics-page), перечисленных выше.



![Страницы поиска по журналам](media/log-analytics-log-search-portals/log-search-portal.png)
