# <a name="tutorial-azure-active-directory-integration-with-spotinst"></a>Руководство по интеграции Azure Active Directory с Spotinst

В этом руководстве описано, как интегрировать Spotinst с Azure Active Directory (Azure AD).

Интеграция Azure AD с приложением Spotinst обеспечивает следующие преимущества.

- С помощью Azure AD вы можете контролировать доступ к Spotinst.
- Вы можете включить автоматический вход пользователей в Spotinst (единый вход) с учетной записью Azure AD.
- Вы можете управлять учетными записями централизованно — на портале Azure.

Подробнее узнать об интеграции приложений SaaS с Azure AD можно в разделе [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory](../manage-apps/what-is-single-sign-on.md).

## <a name="prerequisites"></a>Предварительные требования

Чтобы настроить интеграцию Azure AD с Spotinst, вам потребуется:

- подписка Azure AD;
- подписка Spotinst с поддержкой единого входа.

> [!NOTE]
> Мы не рекомендуем использовать рабочую среду для проверки действий в этом учебнике.

При проверке действий в этом учебнике соблюдайте следующие рекомендации:

- Не используйте рабочую среду без необходимости.
- Если у вас нет пробной среды Azure AD, вы можете [получить пробную версию на один месяц](https://azure.microsoft.com/pricing/free-trial/).

## <a name="scenario-description"></a>Описание сценария
В рамках этого руководства проводится проверка единого входа Azure AD в тестовой среде. Сценарий, описанный в этом учебнике, состоит из двух стандартных блоков.

1. Добавление Spotinst из коллекции.
2. настройка и проверка единого входа в Azure AD.

## <a name="adding-spotinst-from-the-gallery"></a>Добавление Spotinst из коллекции
Чтобы настроить интеграцию Spotinst с Azure AD, необходимо добавить Spotinst из коллекции в список управляемых приложений SaaS.

**Чтобы добавить Spotinst из коллекции, выполните следующие действия.**

1. На **[портале Azure](https://portal.azure.com)** в области навигации слева щелкните значок **Azure Active Directory**. 

    ![Кнопка "Azure Active Directory"][1]

2. Перейдите к разделу **Корпоративные приложения**. Затем выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"][2]
    
3. Чтобы добавить новое приложение, в верхней части диалогового окна нажмите кнопку **Создать приложение**.

    ![Кнопка "Новое приложение"][3]

4. В поле поиска введите **Spotinst**, выберите **Spotinst** на панели результатов и нажмите кнопку **Добавить**, чтобы добавить это приложение.

    ![Spotinst в списке результатов](./media/spotinst-tutorial/tutorial_spotinst_addfromgallery.png)

## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

В этом разделе описана настройка и проверка единого входа Azure AD в Spotinst с использованием тестового пользователя Britta Simon.

Для работы единого входа в Azure AD необходимо знать, какой пользователь в Spotinst соответствует пользователю в Azure AD. Иными словами, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Spotinst.

Чтобы настроить и проверить единый вход Azure AD в Spotinst, вам потребуется выполнить действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-single-sign-on)** необходима, чтобы пользователи могли использовать эту функцию.
2. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
3. **[Создание тестового пользователя Spotinst](#create-a-spotinst-test-user)** требуется для того, чтобы в Spotinst существовал пользователь Britta Simon, связанный с одноименным пользователем в Azure AD.
4. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить Britta Simon использовать единый вход Azure AD.
5. **[Проверка единого входа](#test-single-sign-on)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="configure-azure-ad-single-sign-on"></a>Настройка единого входа Azure AD

В этом разделе описано, как включить единый вход Azure AD на портале Azure и настроить его в приложении Spotinst.

**Чтобы настроить единый вход Azure AD в Spotinst, выполните следующие действия.**

1. На портале Azure на странице интеграции с приложением **Spotinst** щелкните **Единый вход**.

    ![Ссылка "Настройка единого входа"][4]

2. В диалоговом окне **Единый вход** в разделе **Режим** выберите **Вход на основе SAML**, чтобы включить функцию единого входа.
 
    ![Диалоговое окно "Единый вход"](./media/spotinst-tutorial/tutorial_spotinst_samlbase.png)

3. Если вы хотите настроить приложение в режиме, инициируемом поставщиком удостоверений, в разделе**Домены и URL-адреса приложения Spotinst** выполните следующие действия.

    ![Сведения о домене и URL-адресах единого входа для приложения Spotinst](./media/spotinst-tutorial/tutorial_spotinst_url1.png)

    a. Установите флажок **Показать дополнительные параметры URL-адресов**,

    b. В текстовом поле **Состояние ретрансляции** введите `<ID>`.

    c. Если вы хотите настроить приложение в **режиме, инициируемом поставщиком услуг**, тогда в текстовом поле **URL-адрес входа** введите URL-адрес `https://console.spotinst.com`.

    > [!NOTE]
    > Значение "Состояние ретранслятора" приведено для примера. Это значение нужно заменить фактическим значением состояния ретранслятора, которое описано далее в этом руководстве.

4. Приложение Spotinst ожидает утверждения SAML в определенном формате. Настройте следующие утверждения для этого приложения. Управлять значениями этих атрибутов можно в разделе **Атрибуты пользователя** на странице интеграции приложения. На следующем снимке экрана приведен пример.

    ![Настройка единого входа](./media/spotinst-tutorial/tutorial_Spotinst_attribute.png)

5. В разделе **Атрибуты пользователя** диалогового окна **Единый вход** настройте атрибут токена SAML, как показано на рисунке выше, и выполните следующие действия:

    | Имя атрибута | Значение атрибута |
    | ---------------| --------------- |
    | Email | user.mail |
    | FirstName | user.givenname |
    | LastName | user.surname |
    
    a. Щелкните **Добавить атрибут**, чтобы открыть диалоговое окно **Добавление атрибута**.

    ![Настройка единого входа](./media/spotinst-tutorial/tutorial_attribute_04.png)

    ![Настройка единого входа](./media/spotinst-tutorial/tutorial_attribute_05.png)

    b. В текстовом поле **Имя** введите имя атрибута, отображаемое для этой строки.

    c. В списке **Значение** выберите значение атрибута, отображаемое для этой строки.

    d. Оставьте пустым поле **Пространство имен**.

    д. Нажмите кнопку **ОК**.

6. В разделе **Сертификат подписи SAML** щелкните **Metadata XML** (Метаданные XML) и сохраните файл метаданных на компьютере.

    ![Ссылка для скачивания сертификата](./media/spotinst-tutorial/tutorial_spotinst_certificate.png) 

7. Нажмите кнопку **Сохранить** .

    ![Кнопка "Сохранить" в окне настройки единого входа](./media/spotinst-tutorial/tutorial_general_400.png)

8. В другом окне браузера войдите в приложение Spotinst с правами администратора безопасности.

9. Щелкните **значок пользователя** в верхней правой части экрана и выберите **Settings** (Параметры).

    ![Параметры Spotinst](./media/spotinst-tutorial/tutorial_spotinst_settings.png)

10. Щелкните вкладку **SECURITY** (Безопасность) вкладке в верхней части экрана, а затем выберите **Identity Providers** (Поставщики удостоверений) и выполните следующие действия.

    ![Безопасность Spotinst](./media/spotinst-tutorial/tutorial_spotinst_security.png)

    a. Скопируйте значение **Relay State** (Состояние ретранслятора) и вставьте его в текстовое поле **Состояние ретранслятора** в разделе **Домены и URL-адреса приложения Spotinst** на портале Azure.

    b. Затем щелкните **BROWSE** (Обзор), чтобы передать XML-файл метаданных, скачанный с портала Azure.

    c. Щелкните **СОХРАНИТЬ**.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

Цель этого раздела — создать на портале Azure тестового пользователя с именем Britta Simon.

   ![Создание тестового пользователя Azure AD][100]

**Чтобы создать тестового пользователя в Azure AD, выполните следующие действия:**

1. На портале Azure в области слева нажмите кнопку **Azure Active Directory**.

    ![Кнопка "Azure Active Directory"](./media/spotinst-tutorial/create_aaduser_01.png)

2. Чтобы открыть список пользователей, перейдите в раздел **Пользователи и группы** и щелкните **Все пользователи**.

    ![Ссылки "Пользователи и группы" и "Все пользователи"](./media/spotinst-tutorial/create_aaduser_02.png)

3. Чтобы открыть диалоговое окно **Пользователь**, в верхней части диалогового окна **Все пользователи** щелкните **Добавить**.

    ![Кнопка "Добавить"](./media/spotinst-tutorial/create_aaduser_03.png)

4. В диалоговом окне **Пользователь** сделайте следующее.

    ![Диалоговое окно "Пользователь"](./media/spotinst-tutorial/create_aaduser_04.png)

    a. В поле **Имя** введите **BrittaSimon**.

    Б. В поле **Имя пользователя** введите адрес электронной почты для пользователя Britta Simon.

    c. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.

    d. Нажмите кнопку **Создать**.

### <a name="create-a-spotinst-test-user"></a>Создание тестового пользователя Spotinst

Цель этого раздела — создать пользователя с именем Britta Simon в Spotinst.

1. Если вы настроили приложение в режиме, инициируемом **поставщиком услуг**, выполните следующие действия.

   a. В другом окне браузера войдите в приложение Spotinst с правами администратора безопасности.

   b. Щелкните **значок пользователя** в верхней правой части экрана и выберите **Settings** (Параметры).

    ![Параметры Spotinst](./media/spotinst-tutorial/tutorial_spotinst_settings.png)

    c. Щелкните **Users** (Пользователи) и выберите **ADD USER** (Добавить пользователя).

    ![Параметры Spotinst](./media/spotinst-tutorial/adduser1.png)

    d. В разделе "Add User" (Добавление пользователя) выполните следующие действия.

    ![Параметры Spotinst](./media/spotinst-tutorial/adduser2.png)

    * В текстовом поле **Full Name** (Полное имя) введите полное имя, например **BrittaSimon**.

    * В текстовом поле **Email** (Электронная почта) введите адрес электронной почты пользователя, например **brittasimon@contoso.com**.

    * Укажите сведения о своей организации с помощью раскрывающихся списков **Organization Role (Роль в организации), Account Role (Роль учетной записи) и Accounts (Учетные записи)**.

2. Если вы настроили приложение в режиме, инициируемом **поставщиком удостоверений**, тот в этом разделе никакие действия с вашей стороны не потребуются. Приложение Spotinst поддерживает JIT-подготовку. Эта функция включена по умолчанию. Пользователь будет создан при попытке получить доступ к приложению Spotinst (если он еще не создан).

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как разрешить пользователю Britta Simon использовать единый вход Azure, предоставив этому пользователю доступ к Spotinst.

![Назначение роли пользователя][200] 

**Чтобы назначить пользователя Britta Simon приложению Spotinst, выполните следующие действия.**

1. На портале Azure откройте представление приложений, перейдите к представлению каталога, а затем выберите **Корпоративные приложения** и щелкните **Все приложения**.

    ![Назначение пользователя][201] 

2. Из списка приложений выберите **Spotinst**.

    ![Ссылка на Spotinst в списке "Приложения"](./media/spotinst-tutorial/tutorial_spotinst_app.png)  

3. В меню слева выберите **Пользователи и группы**.

    ![Ссылка "Пользователи и группы"][202]

4. Нажмите кнопку **Добавить**. Затем в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Область "Добавление назначения"][203]

5. В диалоговом окне **Пользователи и группы** в списке пользователей выберите **Britta Simon**.

6. В диалоговом окне **Пользователи и группы** нажмите кнопку **Выбрать**.

7. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.
