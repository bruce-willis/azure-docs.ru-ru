# <a name="about-microsoft-identity-platform"></a>Общие сведения о платформе удостоверений Майкрософт

Платформа удостоверений Майкрософт является результатом эволюционного развития платформы для разработчиков и службы идентификации Azure Active Directory (Azure AD). С ее помощью разработчики могут создавать приложения, которые обеспечивают вход с помощью любых удостоверений Майкрософт, получают маркеры для вызова Microsoft Graph, других программных интерфейсов Майкрософт или API, созданных разработчиками. Это полнофункциональная платформа, включающая в себя службу аутентификации, библиотеки с открытым кодом, механизмы регистрации и настройки приложений (через портал разработчиков и API приложения), полную документацию по разработке, примеры кода и другое содержимое, предназначенное для разработчиков. Платформа удостоверений Майкрософт поддерживает стандартные отраслевые протоколы, такие как OAuth 2.0 и OpenID Connect.

До сих пор большинство разработчиков использовали платформу Azure AD версии 1.0 для проверки подлинности удостоверений Azure AD (рабочих и учебных учетных записей), запрашивая маркеры из конечной точки Azure AD версии 1.0 с помощью Библиотеки проверки подлинности Azure AD (ADAL), используя портал Azure для регистрации и настройки приложения, а также API Graph Azure AD для программной конфигурации приложений. Платформа Azure AD версии 1.0 является современной платформой, созданной для работы с корпоративными приложениями.

Чтобы расширить и развить возможности платформы удостоверений Майкрософт, можно аутентифицировать более широкий набор удостоверений Майкрософт (идентификаторы Azure AD, учетные записи Майкрософт например, outlook.com и hotmail.com, а также локальные учетные записи и учетные записи социальных сетей через Azure AD B2C), используя известную конечную точку Azure AD версии 2.0. Здесь вы будете использовать Библиотеку проверки подлинности Майкрософт (MSAL), любую библиотеку OAuth 2.0 с открытым исходным кодом или библиотеку OpenID Connect, портал Azure для регистрации и настройки приложений, а также API Microsoft Graph для программной конфигурации приложений. Обновленная платформа удостоверений Майкрософт (в частности, библиотеки MSAL и актуальная процедура регистрации приложения на портале Azure) значительно изменилась за последний год. Чтобы завершить работу с этим выпуском, мы рекомендуем разрабатывать и тестировать приложения, используя последнюю версию платформы удостоверений Майкрософт.

Приложения, использующие последние версии ADAL и MSAL, будут поддерживать совместный единый вход. Приложения, обновленные с ADAL до MSAL, будут поддерживать состояние входа пользователя в систему. Разработчики могут обновлять приложения MSAL по своему усмотрению, так как приложения, созданные с помощью ADAL, по-прежнему будут работать и поддерживаться.

## <a name="microsoft-identity-platform-experience"></a>Работа с платформой удостоверений Майкрософт

На следующей схеме показана общая процедура идентификации Майкрософт, включая процесс регистрации приложений, пакеты SDK, конечные точки и поддерживаемые удостоверения.

![Платформа удостоверений Майкрософт сегодня](./media/about-microsoft-identity-platform/microsoft-identity-platform-today.png)

На платформе удостоверений Майкрософт есть две конечные точки (версии 1.0 и 2.0) и два набора клиентских библиотек для обработки этих конечных точек. При разработке нового приложения рассмотрите преимущества и текущее состояние конечных точек и библиотек проверки подлинности. Рассмотрим также следующее:

* Поддерживаемые платформы

    * [ADAL](active-directory-authentication-libraries.md) поддерживает .NET, JavaScript, iOS, Android, Java и Python.
    * [Предварительная версия MSAL](reference-v2-libraries.md) поддерживает .NET, JavaScript, iOS и Android.
    * Обе конечные точки поддерживают ПО промежуточного слоя сервера .NET и Node.js для защиты API-интерфейсов и входа. 

* Основная часть инноваций, таких как динамическое и добавочное согласие, реализуются в конечной точке версии 2.0 и MSAL. Тем временем мы продолжаем поддерживать версию 1.0 и ADAL.

    Раньше на портале Azure приходилось статически идентифицировать все области, необходимые для приложения. При использовании конечной точки версии 2.0 и порталов, связанных с этой конечной точкой, по-прежнему можно статически определить области. Вы также можете запрашивать их динамически по мере необходимости разрешений для приложения. Благодаря динамическому запросу доступна еще одна дополнительная возможность — добавочное согласие. Добавочное согласие позволяет запрашивать необходимое подмножество областей, когда пользователь впервые выполняет проверку подлинности. При необходимости можно также запрашивать дополнительные области. 
    
    Например, при использовании приложения камеры на мобильном устройстве пользователю будет предложено разрешить приложению доступ к камере, и, только предоставив согласие, пользователь сможет открыть камеру и сделать снимок.  При сохранении снимка в приложении может потребоваться разрешение на чтение и запись фотографий. 

* Возможные критические изменения

    MSAL подходит для использования в рабочей среде. Для этих библиотек предоставляется такой же уровень поддержки MSAL, как и для текущих рабочих библиотек. Во время действия предварительной версии мы можем вносить изменения в API, формат внутреннего кэша и другие механизмы этой библиотеки. Вы будете применять их вместе с исправлениями ошибок или обновлениями функций. Это может повлиять на приложение. Например, изменение формата кэша может повлиять на пользователей (в частности, потребуется повторный вход). Изменение API может потребовать обновления кода. После выпуска общедоступной версии вам понадобится перейти на нее в течение шести месяцев, так как приложения, написанные с помощью предварительной версии библиотеки, могут перестать работать.
