# <a name="create-a-dynamic-group-and-check-status"></a>Создание динамической группы и проверка состояния

В Azure Active Directory (Azure AD) вы можете создавать группы, применяя правило для определения членства на основе свойств пользователя или устройства. Когда атрибуты пользователя или устройства изменяются, Azure AD оценивает все правила динамической группы в клиенте Azure AD и выполняет добавление или удаление. Если пользователь или устройство соответствует правилу группы, они добавляются как участники. Когда они перестают ему соответствовать, они удаляются.

В этой статье описывается, как настроить правило на портале Azure для динамического членства в группах безопасности или группах Office 365. Примеры синтаксиса правил и полный список поддерживаемых свойств, операторов и значений для правил членства см. в статье [Создание динамических групп с членством на основе атрибутов в Azure Active Directory](groups-dynamic-membership.md).

## <a name="to-create-a-group-membership-rule"></a>Создание правила членства в группе

1. Войдите в [центр администрирования Azure AD](https://aad.portal.azure.com) с помощью учетной записи пользователя, назначенной роли глобального администратора, администратора службы Intune или администратора учетных записей пользователей в своем клиенте.
2. Выберите **Группы**.
3. Выберите **Все группы**, а затем — **Новая группа**.

   ![Добавление новой группы](./media/groups-create-rule/new-group-creation.png)

4. В колонке **Группа** введите имя и описание новой группы. Для параметра **Тип членства** выберите значение **Динамический пользователь** или **Динамическое устройство** в зависимости от того, требуется создать правило для пользователей либо устройств, а затем щелкните **Добавить динамический запрос**. Вы можете использовать конструктор правил, чтобы создать простое правило, или самостоятельно создать правило членства. Эта статья содержит дополнительные сведения о доступных атрибутах пользователей и устройств, а также примеры правил членства.

   ![Добавление правила динамического членства](./media/groups-create-rule/add-dynamic-group-rule.png)

5. После создания правила выберите **Добавить запрос** в нижней части колонки.
6. Щелкните **Создать** в колонке **Группа**, чтобы создать группу.

> [!TIP]
> Создание группы завершается неудачей, если указанное правило неправильно сформировано или недопустимо. Уведомление отображается в правом верхнем углу портала. В нем содержится объяснение, почему правило невозможно обработать. Внимательно прочите его, чтобы понять, как нужно изменить правило, чтобы оно стало допустимым.

## <a name="check-processing-status-for-a-membership-rule"></a>Проверка состояния обработки правила членства

Вы можете просматривать состояние обработки членства и дату последнего обновления на странице **Обзор** для группы.
  
  ![Отображение состояния динамической группы](./media/groups-create-rule/group-status.png)

Для состояния **Обработка членства** могут отображаться следующие сообщения о состоянии:

* **Оценка** — изменение группы получено и оценивается.
* **Обработка** — обновления обрабатываются.
* **Обновление выполнено** — обработка завершена, все применимые обновления внесены.
* **Ошибка обработки** — произошла ошибка при оценке правила членства, не удалось выполнить обработку.
* **Обновление приостановлено** — обновление правила динамического членства приостановлено администратором. MembershipRuleProcessingState имеет статус "Приостановлено".

Для состояния **Последнее обновление членства** могут отображаться следующие сообщения о состоянии:

* &lt;**Дата и время**&gt; — время последнего обновления членства.
* **Выполняется** — обновления выполняются.
* **Неизвестно** — не удается получить время последнего обновления. Это может быть связано с тем, что группа недавно создана.

Если произошла ошибка при обработке правила членства для конкретной группы, отображается предупреждение в верхней части **страницы "Обзор"** для группы. Если более 24 часов невозможно было обработать никакие ожидающие обработки изменения динамического членства для всех групп в клиенте, отображается предупреждение в верхней части страницы **Все группы**.

![сообщение об ошибке обработки](./media/groups-create-rule/processing-error.png)
