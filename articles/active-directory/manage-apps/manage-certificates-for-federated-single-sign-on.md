# <a name="manage-certificates-for-federated-single-sign-on-in-azure-active-directory"></a>Управление сертификатами для федеративного единого входа в Azure Active Directory
В этой статье рассматриваются распространенные вопросы и предоставляются сведения, связанные с сертификатами, создаваемыми службой Azure Active Directory (Azure AD) для обеспечения федеративного единого входа в приложения SaaS. Эти приложения можно добавить из коллекции приложений Azure AD или с помощью шаблона приложения не из коллекции. Настройте приложение для федеративного единого входа.

Эта статья относится только к тем приложениям, которые настроены для использования единого входа Azure AD с федерацией SAML, как показано в следующем примере.

![единого входа Azure AD](./media/manage-certificates-for-federated-single-sign-on/saml_sso.PNG)

## <a name="auto-generated-certificate-for-gallery-and-non-gallery-applications"></a>Автоматически созданный сертификат для приложений из коллекции и не из коллекции
При добавлении нового приложения из коллекции и настройке входа на основе SAML служба Azure AD создает для этого приложения сертификат со сроком действия 3 года. Этот сертификат можно скачать в разделе **Сертификат подписи SAML**. Для приложений из коллекции в этом разделе может предлагаться скачать сертификат или метаданные федерации в зависимости от требований приложения.

![Единый вход Azure AD](./media/manage-certificates-for-federated-single-sign-on/saml_certificate_download.png)

## <a name="customize-the-expiration-date-for-your-federation-certificate-and-roll-it-over-to-a-new-certificate"></a>Настройка даты окончания срока действия сертификата федерации и его замена новым сертификатом
По умолчанию срок действия сертификатов истекает через три года. Вы можете выбрать другой срок действия сертификата, выполнив следующие действия.
На снимках экрана для примера используется Salesforce, но эти действия можно применять для любого федеративного приложения SaaS.

1. На [портале Azure](https://aad.portal.azure.com) щелкните **Корпоративные приложения** в левой области, а затем на странице **Обзор** щелкните **Новое приложение**.

   ![Открытие мастера настройки единого входа](./media/manage-certificates-for-federated-single-sign-on/enterprise_application_new_application.png)

2. Выполните поиск в коллекции и выберите то приложение, которое необходимо добавить. Если не удается найти необходимое приложение, то добавьте его с помощью параметра **Приложение не из коллекции**. Эта функция доступна только для номера SKU Azure AD Premium (P1 и P2).

    ![Единый вход Azure AD](./media/manage-certificates-for-federated-single-sign-on/add_gallery_application.png)

3. Щелкните ссылку **Единый вход** в левой области навигации и замените **Режим единого входа** на **Вход на основе SAML**. При этом для вашего приложения будет создан сертификат со сроком действия 3 года.

4. Чтобы создать сертификат, в разделе **Сертификат подписи SAML** щелкните ссылку **Создать сертификат**.

    ![Создание нового сертификата](./media/manage-certificates-for-federated-single-sign-on/create_new_certficate.png)

5. Ссылка **Создать сертификат** открывает элемент управления "Календарь". Можно задать любые дату и время до трех лет с текущей даты. Выбранные дата и время определят срок действия нового сертификата. Выберите команду **Сохранить**.

    ![Загрузка и отправка сертификата](./media/manage-certificates-for-federated-single-sign-on/certifcate_date_selection.PNG)

6. Теперь новый сертификат доступен для скачивания. Щелкните ссылку **Сертификат**, чтобы скачать его. На этом этапе сертификат еще не активен. Когда требуется перейти на этот сертификат, установите флажок **Make new certificate active** (Сделать новый сертификат активным) и нажмите кнопку **Сохранить**. С этого момента служба Azure AD начнет использовать новый сертификат для подписания ответов.

7.  Чтобы узнать, как передать сертификат в определенное приложение SaaS, щелкните на странице ссылку **View application configuration tutorial** (Просмотреть руководство по настройке приложения).

## <a name="certificate-expiration-notification-email"></a>Уведомление по электронной почте об истечении срока действия сертификата

Azure AD будет отправлять сообщение электронной почты с уведомлением за 60, 30 и 7 дней до истечения срока действия сертификата SAML. Чтобы указать адрес электронной почты, на который будет отправлено уведомление, сделайте следующее:

- В приложении Azure Active Directory на странице единого входа щелкните поле "Сообщение электронной почты с уведомлением".
- Введите адрес электронной почты, на который будут отправляться сообщения электронной почты с уведомлением об истечении срока действия сертификата. По умолчанию в этом поле используется адрес электронной почты администратора, добавившего приложение.

Вы получите сообщение электронной почты с уведомлением от aadnotification@microsoft.com. Во избежание попадания сообщения электронной почты с уведомлением в папку нежелательной почты убедитесь, что этот адрес электронной почты добавлен в список ваших контактов. 

## <a name="renew-a-certificate-that-will-soon-expire"></a>Обновление сертификата с истекающим сроком действия
Описанный ниже процесс обновления не приводит к значительному простою. Приведенные в этом разделе снимки экрана описывают использование Salesforce в качестве примера. Эти действия можно применить к любому федеративному приложению SaaS.

1. На странице **Единый вход** приложения **Azure Active Directory** создайте сертификат для своего приложения. Для этого в разделе **Сертификат подписи SAML** щелкните ссылку **Создать сертификат**.

    ![Создание нового сертификата](./media/manage-certificates-for-federated-single-sign-on/create_new_certficate.png)

2. Выберите дату и время истечения срока действия нового сертификата и нажмите кнопку **Сохранить**.

3. Скачайте сертификат в разделе **Сертификат подписи SAML**. Передайте новый сертификат на экране настройки единого входа приложения SaaS. Чтобы узнать, как это сделать для определенного приложения SaaS, щелкните ссылку **View application configuration tutorial** (Просмотреть руководство по настройке приложения).
   
4. Чтобы активировать новый сертификат в Azure AD, установите флажок **Make new certificate active** (Сделать новый сертификат активным) и нажмите кнопку **Сохранить** в верхней части страницы. При этом на стороне Azure AD активируется новый сертификат. Состояние сертификата меняется с **Новый** на **Активный**. С этого момента служба Azure AD начнет использовать новый сертификат для подписания ответов. 
   
    ![Создание нового сертификата](./media/manage-certificates-for-federated-single-sign-on/new_certificate_download.png)
