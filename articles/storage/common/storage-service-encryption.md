# <a name="azure-storage-service-encryption-for-data-at-rest"></a>Шифрование службы хранилища Azure для неактивных данных
Шифрование службы хранилища Azure для неактивных данных помогает защитить данные в соответствии с применяемыми в вашей организации требованиями к безопасности и соответствию. С помощью этой функции платформа службы хранилища Azure автоматически шифрует данные перед их сохранением в службе "Управляемые диски Azure", хранилище BLOB-объектов Azure, службе файлов Azure или хранилище очередей Azure и расшифровывает данные перед извлечением. Процесс шифрования, шифрования при хранении, расшифровки и управления ключами в шифровании службы хранения является прозрачным для пользователей. При записи всех данных на платформу службы хранилища Azure используется 256-битное [шифрование AES](https://en.wikipedia.org/wiki/Advanced_Encryption_Standard), самая надежная технология блочного шифрования на сегодняшний день.

Шифрование службы хранилища включено для всех новых и имеющихся учетных записей хранения. Его нельзя отключить. Так как данные защищены по умолчанию, вам не нужно изменять код и приложения, чтобы воспользоваться преимуществами шифрования службы хранилища.

Компонент автоматически шифрует данные в следующих расположениях:

- Службы хранилища Azure:
    - Управляемые диски Azure
    - Хранилище больших двоичных объектов Azure
    - Файлы Azure
    - хранилище очередей Azure;
    - хранилище таблиц Azure.  
- На обоих уровнях производительности ("Стандартный" и "Премиум").
- В обоих моделях развертывания (Azure Resource Manager и классической).

Шифрование службы хранилища не влияет на производительность служб хранилища Azure.

Вы можете использовать ключи шифрования с шифрованием службы хранилища, управляемые корпорацией Майкрософт, или собственные ключи шифрования. Дополнительные сведения об использовании собственных ключей см. в статье [Шифрование службы хранилища с помощью управляемых клиентом ключей в Azure Key Vault](storage-service-encryption-customer-managed-keys.md).

## <a name="view-encryption-settings-in-the-azure-portal"></a>Просмотр параметров шифрования на портале Azure
Чтобы просмотреть настройки для шифрования службы хранения, войдите на [портал Azure](https://portal.azure.com) и выберите нужную учетную запись. В области **Параметры** выберите параметр **Шифрование**.

## <a name="faq-for-storage-service-encryption"></a>Часто задаваемые вопросы о шифровании службы хранилища
**Как зашифровать данные в учетной записи хранения Resource Manager?**  
Шифрование службы хранилища включено для всех учетных записей хранения (классических и Resource Manager). Все файлы в учетной записи хранения, созданные до включения шифрования, будут зашифрованы задним числом с помощью фонового процесса шифрования.

**Включена ли функция "Шифрование службы хранилища" по умолчанию при создании учетной записи хранения?**  
Да, функция "Шифрование службы хранилища" включена для всех учетных записей хранения и служб хранилища Azure.

**У меня есть учетная запись Resource Manager. Могу ли я включить в ней шифрование службы хранилища?**  
Шифрование службы хранилища по умолчанию включено для всех имеющихся учетных записей хранения Resource Manager. Данная возможность поддерживается для хранилища BLOB-объектов, службы "Файлы Azure", хранилища очередей Azure и табличного хранилища Azure. 

**Могу ли я отключить шифрование для учетной записи хранения?**  
Шифрование включено по умолчанию, и возможность отключения шифрования для вашей учетной записи не предусмотрена. 

**Насколько дороже обходится служба хранилища Azure с включенной функцией "Шифрование службы хранилища"?**  
Дополнительные затраты не требуются.

**Можно ли использовать собственные ключи шифрования?**  
Да, для хранилища BLOB-объектов Azure и службы файлов Azure можно использовать собственные ключи шифрования. Управляемые клиентом ключи сейчас не поддерживаются в службе "Управляемые диски Azure". Дополнительные сведения см. в статье [Шифрование службы хранилища с помощью управляемых клиентом ключей в Azure Key Vault](storage-service-encryption-customer-managed-keys.md).

**Можно ли отменить доступ к ключам шифрования?**  
Да, если вы [используете собственные ключи шифрования](storage-service-encryption-customer-managed-keys.md) в Azure Key Vault.

**Чем функция "Шифрование службы хранилища" отличается от шифрования дисков Azure?**  
Шифрование дисков Azure обеспечивает интеграцию решений на основе операционной системы, например BitLocker, DM-Crypt и Azure Key Vault. Шифрование службы хранилища обеспечивает встроенное шифрование на уровне платформы службы хранилища Azure (ниже уровня виртуальной машины).

**У меня есть классическая учетная запись хранения. Могу ли я включить в ней шифрование службы хранилища?**  
Шифрование службы хранилища включено для всех учетных записей хранения (классических и Resource Manager).

**Каким образом можно зашифровать данные в классической учетной записи хранения?**  
Если шифрование по умолчанию включено, любые данные, которые хранятся в службах хранилища Azure, автоматически шифруются. 

**Можно ли создавать учетные записи хранения с включенной функцией "Шифрование службы хранилища" с помощью Azure PowerShell и Azure CLI?**  
Шифрование службы хранилища по умолчанию включено во время создания любой учетной записи хранения (классической или учетной записи Resource Manager). Вы можете проверить свойства учетной записи с помощью Azure PowerShell и Azure CLI.

**Для моей учетной записи хранения настроена геоизбыточная репликация. Если включить шифрование службы хранилища, будет зашифрована и избыточная копия?**  
Да, шифрованию подлежат все копии учетной записи хранения. Поддерживаются все варианты избыточности — локально избыточное хранилище, хранилище, избыточное в пределах зоны, геоизбыточное хранилище и геоизбыточное хранилище с доступом для чтения.

**Шифрование службы хранилища разрешено только в определенных регионах?**  
Функция "Шифрование службы хранилища" доступна для всех служб во всех регионах.

**Соответствует ли функция "Шифрование службы хранилища" стандарту FIPS 140-2?**  
Да, она соответствует стандарту FIPS 140-2.

**Куда обратиться, чтобы задать вопрос или оставить отзыв?**  
По любым вопросам, связанным с функцией "Шифрование службы хранилища", или чтобы оставить отзыв, обращайтесь по адресу [ssediscussions@microsoft.com](mailto:ssediscussions@microsoft.com).
