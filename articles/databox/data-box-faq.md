# <a name="azure-data-box-frequently-asked-questions"></a>Часто задаваемые вопросы об Azure Data Box

Гибридное решение Microsoft Azure Data Box позволяет быстро, надежно и недорого передать в Azure много терабайт данных. В этой статье собраны ответы на вопросы, которые могут возникнуть при работе с Data Box на портале Azure. 

Вопросы и ответы упорядочены по следующим категориям:

- Сведения о службе
- Заказ устройства
- Настройка и подключение 
- Отслеживание состояния
- Копирование данных 
- Доставка устройства
- проверка и отправка данных. 
- Поддержка процедур обеспечения сохранности

## <a name="about-the-service"></a>Сведения о службе

### <a name="q-what-is-azure-data-box-service"></a>В. Что такое служба Azure Data Box? 
О.  Служба Azure Data Box предназначена для приема данных в автономном режиме. Эта служба управляет массивом продуктов для хранения с разными объемами, которые предназначены для физической транспортировки данных. 

### <a name="q-what-is-azure-data-box"></a>В. Что такое Azure Data Box?
О. Azure Data Box позволяет быстро, недорого и безопасно передавать терабайты данных в среду Azure и из нее. Устройство Data Box можно заказать на портале Azure. Корпорация Майкрософт отправит вам устройство хранения емкостью 80 ТБ через региональную транспортную службу. 

Получив устройство, вы сможете быстро настроить его через локальный пользовательский веб-интерфейс. Скопируйте данные с серверов на устройство и отправьте его обратно в Azure. В центре обработки данных Azure данные автоматически передаются с устройства в Azure. Весь процесс от начала и до конца контролируется службой Data Box на портале Azure.

### <a name="q-when-should-i-use-data-box"></a>В. Когда следует использовать Data Box?
О. Data Box будет полезен, если у вас есть от 40 до 500 ТБ данных, которые нужно перенести в Azure. Для данных размером менее 40 ТБ используйте Диск Data Box, а для данных размером более 500 ТБ зарегистрируйтесь в службе Data Box Heavy.

### <a name="q-what-is-the-price-of-data-box"></a>В. Какова стоимость Data Box?
О. Базовая стоимость Data Box включает его использование до 10 дней. Выбрав модель продукта при создании заказа на портале Azure, вы увидите стоимость конкретного устройства. Доставка осуществляется бесплатно, но взимается обычная плата за службу хранилища Azure. Дополнительные сведения см. на [странице цен на Azure Data Box](https://azure.microsoft.com/pricing/details/storage/databox/). 

### <a name="q-what-is-the-maximum-amount-of-data-i-can-transfer-with-data-box-in-one-instance"></a>В. Какой максимальный объем данных можно передать за один раз с помощью Data Box?
О. Номинальная емкость Data Box составляет 100 ТБ, доступная емкость — 80 ТБ. Вы можете передать с помощью Data Box не более 80 ТБ данных. Для данных большего объема следует заказать больше устройств.

### <a name="q-how-can-i-check-if-data-box-is-available-in-my-region"></a>В. Как проверить доступность Data Box в моем регионе? 
О.  Сведения о доступности Data Box в разных странах см. на странице [доступности по регионам](data-box-overview.md#region-availability).  

### <a name="q-which-regions-can-i-store-data-in-with-data-box"></a>В. В каких регионах можно сохранять данные с помощью Data Box?
О. Data Box поддерживается во всех регионах США, Западной и Северной Европы, а также во Франции и Великобритании. Поддерживаются регионы только общедоступного облака Azure. Облако Azure для государственных организаций и другие национальные облака не поддерживаются. Дополнительные сведения см. в разделе [Регионы доступности](data-box-overview.md#region-availability).

### <a name="q-whom-should-i-contact-if-i-encounter-any-issues-with-data-box"></a>В. С кем мне связаться, если возникнут проблемы с Data Box?
О. При любых проблемах с Data Box обратитесь в [службу поддержки Майкрософт](data-box-disk-contact-microsoft-support.md).


## <a name="order-device"></a>Заказ устройства

### <a name="q-how-do-i-get-data-box"></a>В. Как получить Data Box? 
О.  Чтобы получить Azure Data Box, войдите на портал Azure и создайте заказ Data Box. Введите свои контактные данные и сведения об уведомлениях. Data Box будет доставлен вам в течение 10 дней с момента размещения заказа в зависимости от доступности. Дополнительные сведения см. в описании [процесса заказа Data Box](data-box-deploy-ordered.md).

### <a name="q-i-was-not-able-to-create-a-data-box-order-in-the-azure-portal-why-would-this-be"></a>В. Я не могу создать заказ Data Box на портале Azure. С чем это связано?
О. Если вам не удалось создать заказ Data Box, значит проблема связана с вашей подпиской или типом доступа. 

Сначала проверьте подписку. Data Box доступен только для подписок с Соглашением Enterprise (EA), поставщика облачных решений (CSP) и с оплатой по мере использования. Если вы используете подписку другого типа, обратитесь в службу поддержки Майкрософт и обновите подписку.

Если же подписка уже относится к поддерживаемому типу, проверьте уровень доступа для подписки. Чтобы создать заказ, нужно иметь роль участника или владельца для подписки.

### <a name="q-i-ordered-a-couple-of-data-box-devices-i-am-not-able-to-create-any-additional-orders-why-would-this-be"></a>В. Заказав несколько устройств Data Box, я не могу создавать новые заказы. С чем это связано?
О. Мы разрешаем не более 5 активных заказов на одну подписку в каждом коммерческом регионе (сочетание страны и региона). Если вам нужно больше устройств, обратитесь в службу поддержки Майкрософт и попросите увеличить лимит для вашей подписки.

### <a name="q-when-i-try-to-create-an-order-i-receive-a-notification-that-the-data-box-service-is-not-available-what-does-this-mean"></a>В. При попытке создать заказ я получаю уведомление, что служба Data Box не доступна. Что это означает?
О. Это означает, что служба Data Box недоступна для выбранного сочетания страны и региона. Изменив это сочетание, скорее всего, вы сможете воспользоваться службой Data Box. Список регионов, где доступна эта служба, см. в описании [доступности Data Box по регионам](data-box-overview.md#region-availability).

### <a name="q-i-placed-my-data-box-order-few-days-back-when-will-i-receive-my-data-box"></a>В. Мой заказ на Data Box размещен несколько дней назад. Когда я получу устройство Data Box?
О. После размещения заказа мы проверяем, есть ли доступное для этого заказа устройство. Если устройство доступно, мы отправим его в течение 10 дней. Но разумеется, бывают периоды повышенного спроса. В такие периоды заказ помещается в очередь, состояние которой вы можете отслеживать на портале Azure. Если заказ не будет выполнен в течение 90 дней, он автоматически отменяется. 

### <a name="q-i-have-filled-up-my-data-box-with-data-and-need-to-order-another-one-is-there-a-way-to-quickly-place-the-order"></a>В. Мой диск Data Box полностью заполнен данными, и я хочу заказать еще один. Есть ли способ быстро разместить заказ?
О. Можно клонировать предыдущий заказ. В этом случае создается тот же заказ, сведения которого можно изменять, только без необходимости вводить сведения об адресе, контактные данные и информацию для уведомлений. 

## <a name="configure-and-connect"></a>Настройка и подключение

### <a name="q-how-do-i-unlock-the-data-box"></a>В. Как разблокировать Data Box? 
О.  На портале Azure перейдите к своему заказу Data Box и перейдите к разделу **Сведения об устройстве**. Скопируйте пароль разблокирования. Используя этот пароль, войдите в локальный пользовательский интерфейс Data Box. Дополнительные сведения см. в руководстве [по распаковке, подсоединению кабелей и подключению Azure Data Box](data-box-deploy-set-up.md).

### <a name="q-can-i-use-a-linux-host-computer-to-connect-and-copy-the-data-on-to-the-data-box"></a>В. Можно ли использовать главный компьютер Linux для подключения Data Box и копирования данных?
О.  Да. Data Box можно подключать к клиентам SMB и NFS. Дополнительные сведения см. в списке [поддерживаемых операционных систем](data-box-system-requirements.md) для главного компьютера.

### <a name="q-my-data-box-is-dispatched-but-now-i-want-to-cancel-this-order-why-is-the-cancel-button-not-available"></a>В. Data Box уже отправлен, но я хочу отменить этот заказ. Почему кнопка "Отмена" недоступна?
О.  Заказ можно отменить только в период от момента заказа и до его обработки. Когда заказ на Data Box уже обработан, его невозможно отменить. 

### <a name="q-can-i-connect-a-data-box-at-the-same-to-multiple-host-computers-to-transfer-data"></a>В. Можно ли подключить Data Box для передачи данных одновременно к нескольким компьютерам?
О. Да. К одному Data Box можно подключить несколько компьютеров для передачи данных, также можно параллельно выполнять несколько заданий копирования. Дополнительные сведения см. в [руководстве по копированию данных на Azure Data Box](data-box-deploy-copy-data.md).

## <a name="track-status"></a>Отслеживание состояния

### <a name="q-how-do-i-track-the-data-box-from-when-i-placed-the-order-to-shipping-the-device-back"></a>В. Как отслеживать Data Box с момента размещения заказа до отправки устройства обратно? 
О.  Состояние заказа на Data Box можно отслеживать на портале Azure. При создании заказа вам также будет предложено предоставить адрес электронной почты для уведомлений. Если он предоставлен, вы будете получать уведомления по почте обо всех изменениях состояния заказа. Дополнительные сведения см. в разделе об [изменении адреса электронной почты для уведомлений](data-box-portal-ui-admin.md#edit-notification-details).

### <a name="q-how-do-i-return-the-device"></a>В. Как вернуть устройство? 
О.  Корпорация Майкрософт выводит транспортную этикетку на дисплей E-ink. Если этикетка не отображается на дисплее E-ink, последовательно выберите **Обзор > Скачать транспортную этикетку**. Скачайте, распечатайте и поместите этикетку в прозрачный пластиковый конверт на устройстве. Затем передайте устройство перевозчику. 

### <a name="q-i-received-an-email-notification-that-my-device-has-reached-the-azure-datacenter-how-do-i-find-out-if-the-data-upload-is-in-progress"></a>В. Я получил по электронной почте уведомление о том, что устройство прибыло в центр обработки данных Azure. Как мне узнать, выполняется ли передача данных?
О. Вы можете открыть заказ на Data Box на портале Azure и перейти к разделу **Обзор**. Если передача данных в Azure уже началась, в области справа вы увидите ход выполнения копирования. 

## <a name="migrate-data"></a>Перенос данных

### <a name="q-what-is-the-maximum-data-size-that-can-be-used-with-data-box"></a>В. Какой максимальный размер данных для Data Box?  
О.  Рабочий объем Data Box составляет 80 ТБ. Одно устройство Data Box можно использовать для данных размером от 40 до 80 ТБ. Для более крупных размеров, вплоть до 500 ТБ, следует заказать несколько устройств Data Box. Если же размер данных превышает 500 ТБ, зарегистрируйтесь в службе Data Box Heavy.  

### <a name="q-what-are-the-maximum-block-blob-and-page-blob-sizes-supported-by-data-box"></a>В. Какой максимальный размер блочного и страничного BLOB-объектов поддерживает Data Box? 
О.  Максимальные размеры регулируются ограничениями службы хранилища Azure. Максимальный размер блочного BLOB-объекта составляет приблизительно 4,768 ТиБ, а максимальный размер большого двоичного объекта составляет 8 ТиБ. Дополнительные сведения см. в статье [Целевые показатели масштабируемости и производительности службы хранилища Azure](../storage/common/storage-scalability-targets.md). 

### <a name="q-how-do-i-know-that-my-data-is-secure-during-transit"></a>В. Как узнать, что мои данные защищены во время передачи? 
О. Мы применяем несколько механизмов безопасности, защищающих Data Box на этапе транспортировки. Сюда входят печати с контролем вскрытия, аппаратные и программные средства обнаружения вскрытия, пароль для разблокировки устройства. Дополнительные сведения см. в статье [Azure Data Box security and data protection](data-box-security.md) (Безопасность Azure Data Box и защита данных).

### <a name="q-how-do-i-copy-the-data-to-the-data-box"></a>В. Как скопировать данные на Data Box? 
О.  Если вы используете клиент SMB, данные можно скопировать на устройство любым средством копирования SMB: Robocopy, Diskboss или просто проводник Windows. 

Если вы используете клиент NFS, воспользуйтесь средствами [rsync](https://rsync.samba.org/), [FreeFileSync](https://www.freefilesync.org/), [Unison](https://www.cis.upenn.edu/~bcpierce/unison/) или [Ultracopier](https://ultracopier.first-world.info/). 

Дополнительные сведения см. в [руководстве по копированию данных на Azure Data Box](data-box-deploy-copy-data.md).

### <a name="q-are-there-any-tips-to-speed-up-the-data-copy"></a>В. Есть ли советы по ускорению копирования данных?
О.  Чтобы ускорить процесс копирования:

- Используйте несколько потоков для копирования данных. Например, с Robocopy используйте многопоточный вариант. Дополнительные сведения об используемой команде см. в [руководстве по копированию данных на диск Azure Data Box и их проверке](data-box-deploy-copy-data.md).
- Используйте несколько сеансов.
- Вместо копирования из сетевой папки (где вы можете быть ограничены скоростью сети) копируйте данные, расположенные локально на компьютере, к которому подключен Data Box.
- Оцените производительность компьютера, используемого для копирования данных. Скачайте [средство Bluestop FIO](https://bluestop.org/fio/) и используйте его, чтобы измерять производительность оборудования сервера.

## <a name="ship-device"></a>Доставка устройства

<!--### Q. How do I schedule a pickup for my Data Box?--> 

### <a name="q-my-device-was-delivered-but-the-device-seems-to-be-damaged-what-should-i-do"></a>В. Мне пришло устройство, но оно выглядит поврежденным. Что делать?
О. Если устройство прибыло поврежденным или с признаками вскрытия, не используйте такое устройство. Как можно быстрее [обратитесь в службу поддержки Майкрософт](data-box-disk-contact-microsoft-support.md) и верните устройство. Вы также можете создать новый заказ на Data Box, чтобы получить другое устройство. В такой ситуации плата за замену устройства не взимается.

### <a name="q-can-i-use-my-own-shipping-carrier-to-ship-data-box"></a>В. Могу ли я воспользоваться услугами своего перевозчика для отправки Data Box?
О. Для службы Data Box корпорация Майкрософт самостоятельно организует доставку от центра обработки данных Azure и обратно. Если вы хотите воспользоваться услугами своего перевозчика, обратитесь к службе импорта и экспорта Azure. Дополнительные сведения о службе импорта и экспорта Azure см. в [этой статье](../storage/common/storage-import-export-service.md).

### <a name="q-e-ink-display-is-not-showing-the-return-shipment-label-what-should-i-do"></a>В. На дисплее E-ink не отображается транспортная этикетка для обратной доставки. Что делать?
О. Если на дисплее E-ink не отображается транспортная этикетка для обратной доставки, выполните следующие действия:
- Удалите старую транспортную этикетку и любые наклейки, сохранившиеся от предыдущих доставок.
- Перейдите к странице заказа на портале Azure. Выберите **Обзор** и **Скачать транспортную этикетку**. Дополнительные сведения см. в разделе о [скачивании транспортной этикетки](data-box-portal-admin.md#download-shipping-label).
- Распечатайте эту транспортную этикетку и поместите ее в прозрачный пластиковый конверт, прикрепленный к устройству. 
- Проконтролируйте, чтобы этикетка была отчетливо видна. 

### <a name="q-how-is-my-data-protected-during-transit"></a>В. Как мои данные защищаются во время передачи? 
О.  Во время транспортировки защиту данных на Data Box обеспечивают следующие механизмы.
 - Все диски Data Box шифруются по стандарту AES 256 бит. 
 - Устройство блокируется и доступ к данным возможен только после ввода пароля разблокировки.
Дополнительные сведения см. в статье [о функциях безопасности для Data Box](data-box-security.md).  

### <a name="q-i-have-finished-prepare-to-ship-and-shut-down-the-device-can-i-still-add-more-data-to-data-box"></a>В. Процесс подготовки к отправке завершен и устройство отключено. Можно ли после этого добавлять новые данные на Data Box?
О. Да. Вы можете включить устройство и добавить дополнительные данные. Но после этого следует снова запустить процесс **подготовки к отправке**.
  

## <a name="verify-and-upload"></a>Проверка и отправка данных

### <a name="q-how-soon-can-i-access-my-data-in-azure-once-ive-shipped-the-data-box-back"></a>В. Как скоро я получу доступ в Azure к своим данным, отправленным на Data Box? 
О.  Как только для **Копирование данных** будет отображаться состояние **Завершено**, вы сможете использовать свои данные.

### <a name="q-where-is-my-data-located-in-azure-after-the-upload"></a>В. Где располагаются мои данные в Azure после передачи?
О.  При копировании через Data Box данные передаются в одно из следующих расположений в учетной записи хранения Azure в зависимости от формата данных: блочный BLOB-объект, страничный BLOB-объект или файлы Azure.
 - `https://<storage_account_name>.blob.core.windows.net/<containername>` 
 -  `https://<storage_account_name>.file.core.windows.net/<sharename>`
 
 Кроме того, вы можете перейти к этим данным из учетной записи хранения Azure на портале Azure.

### <a name="q-i-just-noticed-that-i-did-not-follow-the-azure-naming-requirements-for-my-containers-will-my-data-fail-to-upload-to-azure"></a>В. Как оказалось, требования к именованию Azure для контейнеров не были соблюдены. Будут ли мои данные отправлены в Azure?
О.  Если в именах контейнеров есть прописные буквы, они автоматически преобразуются в нижний регистр. Если имена не совместимы иным образом (специальные символы, другие языки и т. д.), передача завершается ошибкой. Дополнительные рекомендации по именованию общих папок, контейнеров и файлов см. в статьях: 
- [Именование общих папок и ссылка на них](https://docs.microsoft.com/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata)
- [Соглашения по блочным и страничным BLOB-объектам](https://docs.microsoft.com/rest/api/storageservices/understanding-block-blobs--append-blobs--and-page-blobs).

### <a name="q-how-do-i-verify-the-data-i-copied-onto-data-box"></a>В. Как мне проверить данные, скопированные на Data Box?
О.  Данные автоматически проверяются после копирования, когда вы запускаете процесс **Подготовка к отправке**. В процессе этой проверки Data Box создает список файлов и контрольные суммы. Вы можете скачать этот список файлов и сравнить его с исходными файлами. Дополнительные сведения см. в разделе о [подготовке к отправке](data-box-deploy-copy-data.md#prepare-to-ship).

### <a name="q-what-happens-to-my-data-after-i-have-returned-the-data-box"></a>В. Что произойдет с данными после возвращения Data Box?
О.  Когда завершится копирование данных в Azure, все данные безопасно удаляются с Data Box с соблюдением рекомендаций NIST SP 800-88 в редакции 1. Дополнительные сведения см. в статье [Стирание данных из Data Box](data-box-deploy-picked-up.md#erasure-of-data-from-data-box).

## <a name="audit-report"></a>Отчет аудита

### <a name="how-does-azure-data-box-service-help-support-customers-chain-of-custody-procedure"></a>Как служба Azure Data Box помогает соблюдать клиентские процедуры обеспечения сохранности?
О.  Служба Azure Data Box предоставляет встроенные отчеты, которые можно использовать в документации по обеспечению сохранности. Журналы аудита и копирования доступны в учетной записи хранения в Azure, а полную историю заказа можно скачать со страницы заказа на портале Azure после выполнения заказа.


### <a name="what-type-of-reporting-is-available-to-support-chain-of-custody"></a>Какие доступны отчеты для поддержки обеспечения сохранности?
О.  Для поддержки обеспечения сохранности предоставляются следующие отчеты:

- логистика транспортировки от DHL и UPS;
- журналы включения и доступа пользователей;
- файл манифеста с данными 64-разрядной циклической проверки избыточности (CRC-64) или контрольными суммами для каждого файла, успешно сохраненного на Data Box;
- отчеты о файлах, которые не удалось отправить в учетную запись хранения Azure;
- протокол очистки устройства Data Box (в соответствии со стандартом NIST 800 88R1) после копирования данных в учетную запись хранения Azure.

### <a name="are-the-carrier-tracking-logs--from-upsdhl-available"></a>Доступны ли журналы отслеживания от перевозчика (UPS или DHL)? 
О.  Журналы отслеживания перевозчика включаются в журнал аудита Data Box. Этот отчет будет вам доступен после того, как устройство вернется в центр обработки данных Azure и будет выполнена очистка его дисков. Если возникнет срочная потребность в этих данных, вы можете получить сведения об отслеживании на веб-сайте перевозчика, указав номер отслеживания.

### <a name="can-i-transport-the-data-box-to-azure-datacenter"></a>Могу ли я сам привезти Data Box в центр обработки данных Azure? 
О.  Нет. В настоящее время центр обработки данных Azure не принимает доставки Data Box от клиентов или других перевозчиков, кроме UPS или DHL.
